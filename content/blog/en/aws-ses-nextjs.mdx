## Introduction

Email sending is an essential feature in web applications. This article explains how to implement email functionality using Next.js App Router and AWS SES (Simple Email Service).

## What is AWS SES?

Amazon Simple Email Service (SES) is a cloud-based email sending service provided by AWS. It offers high delivery performance and cost efficiency.

### Key Features

- **High deliverability**: Reliable email delivery leveraging AWS infrastructure
- **Cost efficient**: Pay-as-you-go pricing based on send volume
- **Flexible API**: Supports both REST API and SMTP interfaces

## Implementation Steps

### 1. Install the Package

```bash
npm install @aws-sdk/client-ses
```

### 2. Configure the SES Client

```typescript
import { SESClient, SendEmailCommand } from "@aws-sdk/client-ses";

const sesClient = new SESClient({
  region: process.env.AWS_REGION,
  credentials: {
    accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
    secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
  },
});
```

### 3. Create the API Route

Use Next.js App Router's Route Handler to create an email sending endpoint.

```typescript
export async function POST(request: Request) {
  const { name, email, message } = await request.json();

  const command = new SendEmailCommand({
    Source: "noreply@example.com",
    Destination: {
      ToAddresses: ["admin@example.com"],
    },
    Message: {
      Subject: { Data: `Contact from: ${name}` },
      Body: {
        Text: { Data: `Name: ${name}\nEmail: ${email}\nMessage: ${message}` },
      },
    },
  });

  await sesClient.send(command);
}
```

## Summary

By combining AWS SES with Next.js, you can implement simple yet reliable email functionality. In production, make sure to properly configure rate limiting and error handling.
