# Claude Code June 2026 Updates: Fallback Models, Auto Skill Loading, and Security Hardening

Claude Code shipped several practical improvements in June 2026. Here's what matters for day-to-day development workflows.

## Fallback Models

The new `fallbackModel` setting accepts up to three models tried in order when the primary model is overloaded or unavailable.

```json
{
  "model": "claude-opus-4-8",
  "fallbackModel": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5"
  ]
}
```

For long-running agent tasks or batch processing pipelines, hitting rate limits mid-execution used to mean a hard stop. With fallback models configured, Claude Code recovers automatically without interrupting the task.

## Automatic Skill Loading

Plugins in `.claude/skills` directories are now loaded automatically — no Marketplace installation required. The new `/reload-skills` command re-scans skill directories without restarting Claude Code.

For teams managing custom skills in a shared repo, this means a `git pull` is all it takes to pick up updated skills. No restart, no re-install.

## Glob Support in Deny Rules

Deny rules now accept glob patterns in the tool-name position. Specifying `"*"` denies all tools.

```json
{
  "permissions": {
    "deny": [
      "Bash(*)",
      "Write(public/**)"
    ]
  }
}
```

This is useful in restricted team environments where you want to prevent writes to certain directories or block shell execution entirely while still allowing Claude Code to read and analyze code.

## Stronger Cross-Session Security

Messages relayed via `SendMessage` from other Claude sessions no longer carry user authority. In multi-agent setups, this means subagents can't escalate their permissions by routing commands through another session.

## MessageDisplay Hook

A new `MessageDisplay` hook event lets hooks transform or suppress assistant message text before display. Useful for filtering verbose output or building custom team interfaces on top of Claude Code.

## Vim Mode

In Vim mode, pressing `/` in NORMAL mode now opens reverse history search, matching the behavior of Bash and Zsh vi-mode.

## Takeaway

None of these are headline features, but they address real friction points. Fallback models improve reliability for CI pipelines and agent workflows. Auto skill loading simplifies team onboarding. The security improvements matter if you're running Claude Code in multi-agent or shared environments.

Review your `.claude/settings.json` and consider configuring fallback models if you haven't already.
