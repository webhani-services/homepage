By April 2026, the Model Context Protocol (MCP) has become the de facto standard for connecting LLMs to external tools. New MCP servers are shipping weekly for databases, cloud infrastructure, browser automation, and developer tooling. If you're building AI-powered workflows, understanding MCP is no longer optional.

The reason for MCP's rapid adoption is straightforward: it standardizes how LLMs call external tools. Swap the model, swap the tool — the connection layer stays the same.

## How MCP Works

MCP follows a client-server model. The LLM acts as the MCP client and communicates with MCP servers over JSON-RPC, either via stdio or HTTP.

```
LLM (MCP Client)
    ↓ JSON-RPC over stdio / HTTP
MCP Server
    ↓
External tool (database, API, browser, etc.)
```

MCP servers expose three resource types:

- **Tools**: Functions the LLM can call (run queries, read files, send requests)
- **Resources**: Data the LLM can read (schemas, documentation, configs)
- **Prompts**: Reusable prompt templates

## A Minimal MCP Server in TypeScript

Here's a working example of a read-only PostgreSQL MCP server:

```typescript
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { Pool } from "pg";

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

const server = new Server(
  { name: "postgres-readonly", version: "1.0.0" },
  { capabilities: { tools: {} } }
);

server.setRequestHandler("tools/list", async () => ({
  tools: [
    {
      name: "query",
      description: "Execute a read-only SQL query against the database",
      inputSchema: {
        type: "object",
        properties: {
          sql: { type: "string" },
        },
        required: ["sql"],
      },
    },
  ],
}));

server.setRequestHandler("tools/call", async ({ params }) => {
  if (params.name !== "query") throw new Error("Unknown tool");

  const sql = params.arguments.sql as string;
  if (!/^\s*SELECT\b/i.test(sql)) {
    return { content: [{ type: "text", text: "Only SELECT queries are allowed" }] };
  }

  const result = await pool.query(sql);
  return {
    content: [{ type: "text", text: JSON.stringify(result.rows, null, 2) }],
  };
});

const transport = new StdioServerTransport();
await server.connect(transport);
```

## The MCP Ecosystem in April 2026

The server landscape has grown substantially. Here's a breakdown by category:

**Databases**: PostgreSQL, MySQL, SQLite, MongoDB, Redis, Supabase, PlanetScale

**Cloud Infrastructure**: AWS (EC2, S3, CloudWatch, EKS), Kubernetes pod management and log retrieval, Terraform state inspection

**Browser & Web**: Playwright, Puppeteer, web scraping, form automation

**Developer Tools**: GitHub/GitLab (issues, PRs, actions), Jira, Linear, Slack, Discord

DevOps alone now has 18+ active MCP servers, making LLM integration into CI/CD pipelines a practical engineering choice rather than a research project.

## Using MCP with Claude Code

Claude Code supports MCP servers natively via a configuration file:

```json
// .claude/settings.json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_CONNECTION_STRING": "postgresql://localhost/mydb"
      }
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_***"
      }
    }
  }
}
```

With this in place, Claude Code can inspect the database schema while generating queries, or check GitHub issues while writing fixes — without you copy-pasting context manually.

## Security Considerations

MCP servers expand what an LLM can affect, so access control matters.

- **Least privilege**: Use a read-only database user for query tools
- **Secret management**: Keep credentials in environment variables, never in config files committed to source control
- **Input validation**: Don't pass LLM-generated SQL directly to a database without parsing and validating it first
- **Audit logging**: Log all MCP tool calls with the originating prompt for traceability

## Takeaway

MCP's role in the LLM ecosystem is analogous to what HTTP did for web APIs: it gives the community a shared interface to build against. Tool authors write servers once; agent builders use any server regardless of which model is underneath. That separation of concerns is why the ecosystem is growing fast. If you're building LLM workflows today, integrating via MCP will age better than custom tool implementations.
