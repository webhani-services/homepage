## LLMがLLMを書く時代

Anthropicが公表した数字は、AI開発の現場に対して一つの問いを投げかけています。Claude Codeのコードベースのうち、約90%はClaude Code自身が生成しているという事実です。

これは誇張でも宣伝でもなく、Agentic Codingというパラダイムが「実験フェーズ」から「実務フェーズ」に移行しつつある現実を示しています。

## Agentic Codingとは何か

「AI補完」と「Agentic Coding」の違いは質的なものです。

従来のAI補完（GitHub Copilotの初期モデルなど）は、エンジニアが書いている行の続きを提案するものでした。コードの制御権はエンジニアにあり、AIはあくまで入力補助でした。

Agentic Codingでは、AIがタスクを受け取り、コードベースを解析し、計画を立て、実装し、テストを実行し、失敗した場合は原因を特定して修正します。エンジニアはこのループの外から「何を作るか」を指示し、「どう作るか」の大部分をAIに委任します。

```bash
# Claude Codeによるタスク指示の例
# エンジニアはゴールを記述し、実装の詳細はモデルに委ねる

claude "認証ミドルウェアにJWTリフレッシュトークンのローテーション機能を追加してください。
- アクセストークン有効期限: 15分
- リフレッシュトークン有効期限: 7日間
- データベース: Prisma + PostgreSQL
- 既存のテストスイートに統合すること"
```

## 「難しくて直感に反する」という現実

Anthropicのエンジニアの間でも、Claude Codeの活用は「ボタンを押すだけの魔法」ではないと認識されています。実際に高い成果を得るためには、新しいパターンの習得が必要です。

webhaniのチームが実務で見えてきたポイントをいくつか挙げます。

**1. コンテキストの品質がアウトプットを決める**

LLMへの指示はあいまいであるほど、期待と異なる結果が返ってきます。「ファイル構造」「既存の設計パターン」「制約条件」を明示的に伝えることが重要です。

```markdown
# 良いプロンプトの構造

## 背景
- 使用フレームワーク: Next.js 16 App Router
- 認証ライブラリ: Clerk
- DB: Neon (Serverless Postgres)

## タスク
ユーザーのサブスクリプションステータスをチェックして、
未契約ならば /pricing にリダイレクトするミドルウェアを実装してください。

## 制約
- Edge Runtimeで動作すること (Node.js APIは使用不可)
- 既存の proxy.ts に統合すること
```

**2. 批判的思考の放棄は禁物**

AIが生成したコードをそのままマージすることは、未レビューのプルリクエストをマージするのと同じリスクを持ちます。セキュリティ上の問題、パフォーマンスの非効率、既存パターンとの不整合を見抜く目は、エンジニアが持ち続ける必要があります。

**3. ループの構造を設計する**

「一発で全部やってもらう」より「小さなステップを繰り返す」アプローチの方が、高品質なアウトプットにつながります。実装 → レビュー → 修正指示 → 再実装のサイクルを短く回すことが重要です。

## モデルの競争と選択

2026年3月時点で、主要なAIコーディングモデルの状況は以下のとおりです。

| モデル | SWE-bench Verified | 特徴 |
|---|---|---|
| Claude Opus 4.6 | ~80% | Agentic作業に最適化、自律的なタスク実行 |
| GPT-5.3-Codex | ~77% | Terminal-Bench 2.0でリード |
| Llama 4 Scout | 公開ベンチマーク進行中 | 10Mトークンコンテキスト、オープンソース |

オープンソースモデルの進歩も注目に値します。Meta Llama 4 Scoutは1000万トークンのコンテキストウィンドウを持ち、コード生成・文書処理・分類など多くのタスクで商用モデルと遜色ない結果を出しはじめています。自社インフラで推論を完結させたいチームには現実的な選択肢になってきています。

## エンジニアの役割の再定義

Agentic Codingの普及は、エンジニアを「コードを書く人」から「AIの品質管理者・システム設計者」へとシフトさせます。

具体的には以下のスキルの比重が高まります：

- **アーキテクチャ設計力**: AIが局所的な実装を担う分、全体の構造を正しく設計する責任はエンジニアに残ります
- **コードレビュー能力**: AI生成コードの問題点を素早く見抜く読解力
- **プロンプト設計**: タスクを明確に分解し、適切なコンテキストとともに指示する能力
- **テスト設計**: AIが正しい方向に向かっているかを検証するテストを先に書く能力（TDDとの相性が良い）

## まとめ

Claude CodeがそのコードベースをAIで生成するという事実は、Agentic Codingが実用段階に入ったことを示しています。しかし、これはエンジニアが不要になることを意味しません。

AIの作業品質を保証し、システム全体の整合性を維持し、ビジネス要件を技術仕様に落とし込む能力は、依然としてエンジニアの中核的な価値です。

新しいワークフローを早期に習得し、AIとの協働パターンを自分のものにすることが、今後12〜24ヶ月のエンジニアにとっての競争優位になると考えています。
