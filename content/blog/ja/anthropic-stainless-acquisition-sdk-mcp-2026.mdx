## Stainless とは何か

2022年創業の Stainless は、API 仕様から TypeScript、Python、Go、Java、Kotlin などの SDK を自動生成するツールを提供していました。創業者は Stripe 出身のエンジニアで、同社の SDK 設計の知見をプロダクト化した経緯があります。

Stainless の利用者には Anthropic 自身に加え、OpenAI、Google、Cloudflare が含まれます。競合 AI ラボが共通して依存していたインフラが、今回の買収で Anthropic の内製リソースへ転換されることになります。TechCrunch の報道（2026年5月18日）によると、買収額は 3 億ドル以上とされています。

## 買収の戦略的意図

表面上は「SDK 品質の向上」ですが、含意はより広いです。

**MCP エコシステムの制御:** Stainless は SDK 生成だけでなく、MCP（Model Context Protocol）サーバーの生成にも対応していました。MCP は AI エージェントが外部ツールと連携するための標準プロトコルです。Anthropic が MCP を創案し、その主要な生成ツールも内製化することで、エコシステムの中枢に位置する戦略的意図が読み取れます。

**競合ラボへの影響:** Anthropic は Stainless が提供していたホスト型 SDK ジェネレーターサービスを終了すると発表しています。既存顧客は生成済みの SDK の所有権とカスタマイズ権を引き続き持ちますが、新たな自動生成は受けられなくなります。OpenAI や Google は代替手段を調達する必要があります。

**開発者体験の差別化:** 高品質な SDK は API 採用の摩擦を大幅に下げます。型安全性、エラーハンドリング、ページネーション、再試行ロジックが一貫して実装されているかどうかで、開発者の初期体験が大きく変わります。

## SDK 品質が重要な理由

```typescript
// 低品質の SDK: 手動実装の例
const response = await fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: { "x-api-key": apiKey, "content-type": "application/json" },
  body: JSON.stringify({ model: "claude-opus-4-7", max_tokens: 1024, messages }),
});
if (!response.ok) throw new Error(`HTTP ${response.status}: ${await response.text()}`);

// 生成された高品質 SDK: 型安全なエラーハンドリング
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

try {
  const message = await client.messages.create({
    model: "claude-opus-4-7",
    max_tokens: 1024,
    messages: [{ role: "user", content: "Hello" }],
  });
  console.log(message.content);
} catch (error) {
  if (error instanceof Anthropic.APIError) {
    console.error(`Status: ${error.status}`);
    console.error(`Error: ${error.message}`);
    // error.status は number 型として型推論される
  }
}
```

Stainless が生成する SDK はこのような一貫したパターンを言語をまたいで保証します。Python、Go、Java でも同様のインターフェースを持つため、プロジェクト間での認知的負担が減ります。再試行ロジック、レート制限ヘッダーの処理、ストリーミングの組み立てなども一貫して実装されます。

## MCP サーバー生成への影響

開発者として注目すべきは MCP サーバー自動生成の変化です。MCP サーバーを手動で実装する場合は以下のような構造が必要です:

```typescript
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
} from "@modelcontextprotocol/sdk/types.js";

const server = new Server(
  { name: "example-server", version: "0.1.0" },
  { capabilities: { tools: {} } }
);

server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [
    {
      name: "get_user",
      description: "Get user by ID",
      inputSchema: {
        type: "object",
        properties: { id: { type: "string" } },
        required: ["id"],
      },
    },
  ],
}));
```

Stainless はこれを API 仕様から自動生成することができました。ホスト型サービスの終了後、既存顧客は生成コードを手元に持ちつつ独自にメンテナンスする必要があります。新たに MCP サーバーを生成したい場合は代替ツールを検討する段階に入っています。

## 開発者が今すべきこと

**Stainless 既存ユーザーへ:** 生成済みの SDK とサーバーコードはそのまま使用できます。新機能の追加が必要になった場合、手動でのメンテナンスか代替ツールへの移行を検討してください。現時点では [Speakeasy](https://www.speakeasyapi.dev/) や [Fern](https://buildwithfern.com/) が代替候補として挙がっています。

**Anthropic API ユーザーへ:** 公式 SDK は引き続き Anthropic 内製でメンテナンスされます。SDK 品質の低下を懸念する必要はなく、むしろ向上が期待できます。

**MCP サーバーを構築中の場合:** MCP SDK を使った直接実装に習熟しておくことが重要です。Anthropic が MCP と生成ツールの両方を保有することになったため、今後の公式 MCP 開発ツールの改善が期待されます。

## まとめ

Anthropic による Stainless 買収は、AI ラボ間の競争が「モデル性能」だけでなく「開発者エコシステム」の主導権争いへ移行していることを示しています。SDK と MCP サーバーという見えにくいがクリティカルなインフラを内製化することで、Anthropic は開発者体験の全体を制御する戦略を選択しました。

Anthropic プラットフォームで開発するチームには長期的に SDK 品質の向上を意味し、Stainless を他の用途で活用していたチームには移行計画が必要な時点です。
