## はじめに

Webアプリケーションにおいてメール送信は欠かせない機能の一つです。本記事では、Next.js App RouterとAWS SES（Simple Email Service）を組み合わせたメール送信の実装方法を紹介します。

## AWS SESとは

Amazon Simple Email Service（SES）は、AWSが提供するクラウドベースのメール送信サービスです。高い配信性能とコスト効率を兼ね備えています。

### 主な特徴

- **高い到達率**: AWSのインフラを活用した信頼性の高いメール配信
- **コスト効率**: 送信量に応じた従量課金制
- **柔軟なAPI**: REST APIとSMTPインターフェースの両方に対応

## 実装手順

### 1. パッケージのインストール

```bash
npm install @aws-sdk/client-ses
```

### 2. SESクライアントの設定

```typescript
import { SESClient, SendEmailCommand } from "@aws-sdk/client-ses";

const sesClient = new SESClient({
  region: process.env.AWS_REGION,
  credentials: {
    accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
    secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
  },
});
```

### 3. API Routeの作成

Next.js App Routerの Route Handler を使って、メール送信エンドポイントを作成します。

```typescript
export async function POST(request: Request) {
  const { name, email, message } = await request.json();

  const command = new SendEmailCommand({
    Source: "noreply@example.com",
    Destination: {
      ToAddresses: ["admin@example.com"],
    },
    Message: {
      Subject: { Data: `お問い合わせ: ${name}` },
      Body: {
        Text: { Data: `名前: ${name}\nメール: ${email}\n内容: ${message}` },
      },
    },
  });

  await sesClient.send(command);
}
```

## まとめ

AWS SESとNext.jsを組み合わせることで、シンプルかつ信頼性の高いメール送信機能を実装できます。本番環境では、レート制限やエラーハンドリングも適切に設定しましょう。
