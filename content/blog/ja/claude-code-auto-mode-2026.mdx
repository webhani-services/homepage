Anthropic は 2026 年 3 月、Claude Code に「Auto Mode」をリサーチプレビューとして導入しました。これは Claude が自らの判断で「安全に実行できるアクション」を決定し、ユーザーの確認なしに進める機能です。

## Auto Mode とは

従来の Claude Code では、ファイルの編集やコマンドの実行など、影響範囲が大きい操作には都度ユーザーの承認が必要でした。Auto Mode はこのフローを変え、Claude が事前に設定されたルールと文脈に基づいて、低リスクと判断したアクションを自律的に実行します。

重要なのは「leash（リーシュ）」の存在です。Anthropic は自律性を高めながらも、明確な制限を設けています。破壊的操作（ファイルの削除、外部 API への書き込みなど）は依然として承認が必要です。

## 開発ワークフローへの影響

Auto Mode が最も効果を発揮するのは、反復的なリファクタリングや定型的な実装タスクです。

```typescript
// Claude Code が Auto Mode で自律実行するタスク例:
// - 型エラーの修正
// - import の整理
// - コードフォーマット
// - テストの実行（読み取り専用操作）

// ユーザー承認が引き続き必要なタスク:
// - 新規ファイルの作成
// - 外部 API の呼び出し
// - 環境変数の変更
// - ファイルの削除
```

実際に使ってみると、小さなバグ修正やリファクタリングの流れが大幅にスムーズになります。Claude が「これは安全」と判断したステップを自動的に処理し、判断が難しい箇所だけ人間に委ねる設計です。

## AI エージェントの自律性と信頼

Auto Mode の導入は、AI エージェントの「信頼モデル」について改めて考えさせてくれます。

完全な自律性は効率的ですが、予測不可能なリスクを伴います。一方、すべてのアクションに承認を求めるモデルは安全ですが、開発者の作業を頻繁に分断します。Anthropic のアプローチは段階的な信頼の確立です。

1. **可逆性による分類**: 元に戻せる操作は自動実行、戻せない操作は承認必須
2. **コンテキスト理解**: プロジェクトの構造と過去の操作履歴を参照した判断
3. **透明性の維持**: Auto Mode で実行されたアクションは常にログに記録

## 実践的な活用のポイント

**CLAUDE.md の整備が重要になる**

プロジェクトルートの `CLAUDE.md` に明確なルールを記述することで、Claude の自律判断の精度が上がります。「このディレクトリは変更禁止」「このコマンドは常に実行可能」といった明示的な指示が有効です。

```markdown
<!-- CLAUDE.md の記述例 -->
## Auto Mode ルール

- `/infra` ディレクトリは変更禁止
- `npm test` は常に実行可能
- 環境変数ファイル (.env) は読み取り専用
- 新規ファイル作成は必ず承認を求めること
```

**段階的に信頼範囲を広げる**

最初は Auto Mode の適用範囲を絞り、Claude の判断パターンを観察してから徐々に広げていくアプローチが安全です。一週間ほど使ってみてから、問題のない操作カテゴリを追加していくのが現実的です。

**チーム開発では共通のルール設定を**

複数人で同じリポジトリを扱う場合、`CLAUDE.md` をバージョン管理に含め、Auto Mode の動作を統一することが重要です。チームメンバーごとに異なるルールが適用されると、一貫性のない変更が入り込むリスクがあります。

## リサーチプレビューとしての現在地

現時点では Auto Mode はリサーチプレビューであり、設定の細かな制御はまだ限られています。今後の正式リリースでは、操作カテゴリ別の細かな設定や、プロジェクト単位のルール適用が可能になると予想されます。

また、この機能の登場は AI コーディングエージェント全体のトレンドを反映しています。GitHub Copilot Workspace、Cursor、Devin など各社がエージェントの自律性をどこに設定するかで競い合っており、Anthropic のアプローチは「信頼を段階的に稼ぐ」モデルとして一定の説得力があります。

## まとめ

Claude Code の Auto Mode は、AI エージェントが単なる補助ツールから「信頼できる協働者」に近づく一歩です。完全な自律性ではなく、適切な境界線を設けながら効率を高めるアプローチは、実際の開発現場で受け入れられやすい設計です。

`CLAUDE.md` を整備し、段階的に信頼範囲を広げていく使い方が、現段階では最も実践的です。リサーチプレビューの段階から試しておくことで、正式リリース後にスムーズに活用できる土台が作れます。
