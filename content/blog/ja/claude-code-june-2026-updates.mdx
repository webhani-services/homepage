# Claude Code 2026年6月アップデート：Fallback Model、Skill自動ロード、セキュリティ強化

Claude Codeが2026年6月に複数の実用的なアップデートを受けました。本稿では、開発ワークフローに直接影響するいくつかの変更点を詳しく確認します。

## Fallback Models 設定

`fallbackModel` 設定が追加され、プライマリモデルが過負荷または利用不可のときに試みるモデルを最大3つまで順番に指定できるようになりました。

```json
{
  "model": "claude-opus-4-8",
  "fallbackModel": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5"
  ]
}
```

長時間稼働するエージェントタスクやバッチ処理では、モデルのレート制限に当たることがあります。この設定があれば、処理を中断せずに自動的に代替モデルへフォールバックできます。

## Skillディレクトリの自動ロード

`.claude/skills` ディレクトリ内のPluginが、Marketplaceを経由せずに自動でロードされるようになりました。また、Skillディレクトリをリスタートなしで再スキャンする `/reload-skills` コマンドも追加されています。

チーム内でカスタムSkillを管理している場合、`git pull` 後に再起動不要で即座に最新のSkillが反映される点が実用的です。

## Deny Rule の Glob 対応強化

deny rule の tool-name ポジションで glob パターンが使えるようになりました。`"*"` を指定するとすべてのツールを拒否できます。

```json
{
  "permissions": {
    "deny": [
      "Bash(*)",
      "Write(public/**)"
    ]
  }
}
```

特定のディレクトリへの書き込みやShell実行を制限したいチーム環境での利用が想定されます。

## SendMessage のセキュリティ強化

他のClaude Sessionから `SendMessage` で中継されたメッセージは、ユーザー権限を持たなくなりました。マルチエージェント構成において、サブエージェントが過剰な権限で操作を実行するリスクが低減します。

## MessageDisplay Hook

新しい `MessageDisplay` Hookで、アシスタントのメッセージテキストを表示前に変換または非表示にできるようになりました。ログ出力のフィルタリングや、チーム向けカスタムUIの構築に活用できます。

## Vim Mode 改善

Vim モードの NORMAL モードで `/` キーが逆方向の履歴検索を開くようになり、BashやZshのvi-modeと挙動が揃いました。

## まとめ

今回のアップデートは小粒ですが、実際の開発現場での使い勝手を高めるものが多いです。特にFallback ModelsはCI環境やエージェントタスクの安定性向上に直結します。Skill自動ロードもチーム運用を簡略化します。

設定ファイルを見直してこれらの機能を積極的に活用していきましょう。
