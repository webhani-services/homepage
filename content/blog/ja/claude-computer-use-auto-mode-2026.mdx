Anthropic は 2026 年 3 月 24 日、Claude の大型アップデートを発表しました。Pro・Max プランのユーザーを対象に、Claude Code と Cowork で **Computer Use** 機能が利用可能になり、加えて「**Auto Mode**」の Research Preview も始まっています。

## Computer Use とは何か

Computer Use は Claude がコンピューターの画面を「見て」「操作する」機能です。具体的には以下ができます。

- ファイルのオープン・編集
- 開発ツール（ターミナル、ブラウザ、IDE）の起動と操作
- マウスのクリック・ドラッグ
- スクリーン上のナビゲーション

これまでのツールコールはテキストベースの API 呼び出しが中心でしたが、Computer Use によって Claude は GUI アプリケーションとも直接インタラクションできるようになります。

```bash
# Claude Code でマルチステップタスクを委任する例
# 指示: テストが失敗している原因を特定して修正してください
# → Claude が以下を自律的に実行
# 1. ターミナルでテストを実行
# 2. エラーログを読む
# 3. 対象ファイルを開いて修正
# 4. テストを再実行して確認
```

「指示を出す → Claude が提案する → 人間が実行する」というサイクルが、一部のタスクでは「指示を出す → Claude が実行する」に短縮されます。

## Auto Mode：Claude が自分で判断する

Auto Mode は Claude が「どのアクションを自律的に実行してよいか」を自ら判断するモードです。現在は Research Preview として提供されており、いくつかの制限が設けられています。

Anthropic がこのモードに慎重なのは理由があります。エージェントが誤った判断で取り返しのつかないアクション（ファイル削除、外部 API の実行など）を行うリスクを最小化するためです。

Auto Mode の現在の範囲に適したユースケース：

1. **Lint の実行と自動修正適用**：フォーマットエラーを検出して修正する
2. **テスト実行と失敗のサマリー**：CI のように繰り返しテストを回す
3. **コードベースの探索**：リファクタリング前に関連ファイルを特定する
4. **ドキュメント更新**：コードの変更を検知してコメントや README を更新する

## 開発ワークフローへの影響

webhani では Claude Code を日常的なコーディング支援に活用しています。今回の変更で、エージェントとの協業モデルが変わり始めています。

**メリット**
- 繰り返し作業の自動化（ファイルのリネーム、フォーマット適用など）
- 複数ステップのタスクを一括で委任できる
- テキスト API では操作できない GUI ツールとの連携が可能

**注意点**
- アクション範囲の明示的な指定が重要（意図しないファイル操作を防ぐため）
- 本番環境への直接アクセスは慎重に検討する
- Auto Mode はまだ Research Preview のため、プロダクション用途には早計

## Claude Sonnet 4.6 の能力向上

今回のアップデートでは、コーディング・Computer Use・長文コンテキスト推論・エージェント計画など複数の領域で能力が向上しています。1M トークンコンテキストウィンドウ（Beta）も追加されており、大規模なコードベース全体を一度に渡す分析も現実的になってきました。

## まとめ

Claude の Computer Use と Auto Mode は、AI エージェントが「コード提案ツール」から「実際に作業を実行するアシスタント」へと進化していることを示しています。

現時点では読み取り専用の操作（ファイルの確認、テスト結果の確認など）から始めて、徐々に自動化の範囲を広げていくアプローチが現実的です。Auto Mode の正式リリースに向けて、Anthropic がどのような安全性制御を実装するかは引き続き注目です。
