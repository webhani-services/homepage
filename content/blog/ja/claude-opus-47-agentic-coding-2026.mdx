Anthropic が Claude Opus 4.7 をリリースしました。14 のベンチマーク中 12 で最高スコアを記録し、価格は従来の Opus 4.6 と同じ $5/$25 per million tokens を維持しています。単なる性能向上にとどまらず、Anthropic が推奨する使い方そのものが変わってきた点が注目に値します。

## 「ペアプログラマー」から「委任エンジニア」へ

Anthropic の推奨ガイドラインでは、Opus 4.7 を「ペアプログラマー」ではなく「委任エンジニア（delegated engineer）」として扱うことを明示しています。この違いは実務上大きな意味を持ちます。

ペアプログラマーとしての AI は、人間が指示を出すたびに応答する受け身のツールです。一方、委任エンジニアとしての AI は、明確なゴールとコンテキストを渡せば、途中の判断を自律的に行いながらタスクを完遂します。

新たに追加された `xhigh` effort level はこの思想を体現しています。Claude Code などのエージェント型ワークフローでデフォルト設定として推奨されており、複数ステップにわたるタスクを途中で止まらずに処理し続けます。

## Claude Code の改善点

Claude Opus 4.7 のリリースに合わせ、Claude Code にも重要なアップデートが加わりました。

**Worktree Isolation**

並列コーディングタスクを安全に実行するため、Git Worktree を活用した隔離環境が導入されました。複数のフィーチャーブランチを同時に作業させても、互いの変更が干渉しません。

```bash
# Claude Code が内部的に行う操作（概念的な例）
git worktree add .worktrees/feature-auth -b feature/auth
git worktree add .worktrees/feature-api -b feature/api
# 各 Worktree で独立してタスクを実行
```

**Memory Management**

長時間セッションでのコンテキスト管理が改善されました。セッションをまたいで参照すべき情報は永続メモリに保存し、一時的な作業コンテキストは適切にクリーンアップされます。これにより、大規模なコードベースを扱う長期プロジェクトでも品質が安定します。

## Agentic Coding を実際に活用する

理論より実践です。委任エンジニアとして Claude を効果的に使うためのポイントを整理します。

### タスクの粒度を適切に設定する

細かすぎる指示はペアプログラミングになってしまいます。「このファイルの `getUserById` 関数を修正してください」ではなく、「User 取得の処理で N+1 クエリが発生しています。調査して修正し、テストも更新してください」のような形でゴールを渡します。

```
# 良い例（委任）
タスク: 注文一覧ページのパフォーマンスを改善してください。
現状: 100件の注文を表示する際に200以上のクエリが発生しています。
期待: ページロード時間を500ms以下に抑え、既存のテストがすべてパスすること。

# 避けるべき例（過度な細分化）
1. OrderRepository を開いて
2. findAll メソッドを確認して
3. include を追加して
4. ...
```

### コンテキストを構造化して渡す

AI が自律的に判断するためには、十分なコンテキストが必要です。コードベースの規約、制約条件、関連するドキュメントへの参照を最初にまとめて渡すことで、途中での確認往復を減らせます。

### 検証ステップを明示する

タスクの完了条件を明確にしておくと、AI が自己検証できます。

```
完了条件:
- `npm test` がすべてパスすること
- `npm run typecheck` でエラーがないこと
- 変更対象のエンドポイントのレスポンスタイムが改善されること（before/after を記録）
```

## 懸念点と現実的な評価

Agentic Coding はすべてを自動化するわけではありません。現時点での限界も理解しておくことが重要です。

- **ドメイン知識の補完**: ビジネスロジックや組織固有の制約は、コードから読み取れない部分があります。重要な判断ポイントでは人間のレビューが不可欠です。
- **セキュリティの最終確認**: AI が生成したコードは、セキュリティ観点でのレビューをスキップしてはなりません。
- **依存関係の更新**: ライブラリのメジャーバージョンアップなど、広範な影響を持つ変更は慎重に扱う必要があります。

Claude Mythos（選定パートナー向けプレビュー）がサイバーセキュリティ脆弱性の発見・修正に活用されていることからも分かるように、AI の能力は急速に拡張しています。ただし、それを活かすためのワークフロー設計と人間側の判断力が今後さらに重要になります。
