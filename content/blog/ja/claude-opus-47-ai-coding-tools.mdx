## Claude Opus 4.7 が一般公開

Anthropic が Claude Opus 4.7 を正式リリースしました。前バージョン (Opus 4.6) と比較して、高度なソフトウェアエンジニアリングタスクでの性能が向上しており、特にコードレビューの recall が 10% 以上改善されながら precision は維持されているとされています。

SWE-bench のスコアでは GPT-5.5 と拮抗しており、現時点で最高クラスのコーディング性能を持つモデルの一つです。

## コードレビューにおける変化

コードレビューでの recall 改善は実務上、見逃しが減ることを意味します。

具体的には次のようなシナリオで差が出やすいです。

- 大きな PR でのバグ検出（1,000 行以上の差分）
- 複数ファイルをまたぐリファクタリング後の整合性確認
- セキュリティ上の問題点の指摘

```typescript
// 悪い例：クエリに直接文字列を埋め込んでいる
async function getUser(id: string) {
  const user = await db.query(`SELECT * FROM users WHERE id = ${id}`);
  return user;
}

// Claude Opus 4.7 は SQL Injection リスクとパラメータ化クエリの必要性を
// 大規模な差分の中でも一貫して指摘できるようになっています

// 修正後
async function getUser(id: string) {
  const user = await db.query("SELECT * FROM users WHERE id = $1", [id]);
  return user;
}
```

## Claude Code エコシステムの拡充

Claude Opus 4.7 のリリースと合わせて、Claude Code 周辺のエコシステムも整備が進んでいます。

**Prismatic Skills for Claude Code**
Prismatic が Claude Code 向けのオープンソース Plugin をリリースしました。認証や運用インフラの知識を Claude Code に提供することで、Integration 開発を高速化することを目的としています。一般的なボイラープレートではなく、コンテキストに沿った提案を受け取れるようになります。

**Snyk との連携**
Snyk の AI Security Platform に Claude の Reasoning 機能が統合されました。脆弱性の発見だけでなく、修正の提案もより確度が高くなるとされています。

## Cursor・Windsurf での利用

Claude は Cursor・Windsurf・Claude Code の 3 つの主要な AI コーディングエディタで採用されています。Opus 4.7 が各ツールに反映されることで、日常的な開発作業でも恩恵を受けられます。

ツールを変えなくても、モデルのアップデートがそのまま反映される点はシームレスです。ただし、モデルが変わってもプロンプトの品質が重要な点は変わりません。タスクのコンテキスト、対象ファイルのパス、期待する出力形式を明示することで、パフォーマンスを最大限に引き出せます。

```
# Claude Code での効果的な指示例

「src/api/middleware/auth.ts の JWT 検証ロジックをレビューしてください。
特に以下の点に注目してください：
- トークンの有効期限チェック
- 無効なトークンに対するエラーレスポンス
- リフレッシュフローのエッジケース

問題点を行番号付きで列挙し、具体的な修正案を提示してください。」
```

## webhani の視点

webhani では、Claude Code と Cursor を組み合わせた開発フローを複数のプロジェクトで採用しています。Opus 4.7 の改善は、特に大規模なコードベースでのコードレビューフェーズの品質向上に寄与すると見ています。Opus 4.6 では見逃していたコンテキスト依存のバグを、より確実に捉えられるようになっています。

ただし、AI コーディングアシスタントはエンジニアの判断をサポートするツールです。生成されたコードや提案は必ずレビューし、テストを通すことが前提です。ツールの性能が上がるほど、エンジニア側のレビュースキルも重要になります。
