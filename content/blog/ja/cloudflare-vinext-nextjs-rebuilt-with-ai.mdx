## 1人のエンジニアが1週間でNext.jsを再実装

2026年3月、Cloudflareはvinext（ヴィネクスト）という実験的なプロジェクトを公開しました。これはVite上に構築されたNext.js互換フレームワークで、1人のエンジニアがAIの補助を受けながら1週間・約1,100ドルのAPI費用で開発したものです。

この数字だけで見ると信じがたいですが、実際のコードは以下の品質管理を通過しています：

- Vitestによるユニットテスト：1,700件以上
- PlaywrightによるE2Eテスト：380件
- TypeScript型チェック通過
- ESLintクリーン

そして早期ベンチマークではNext.jsに対して**4.4倍高速なビルド速度**を記録しています。

## 何がこれを可能にしたか

### AIセッションの規模

開発中に使用されたAIセッションは800以上。ほぼすべてのコードがAIとのセッションから生まれています。エンジニアは「AIに書かせ、自分でレビュー・品質管理する」というワークフローを徹底しました。

### Viteという土台

Next.jsのビルドシステムはwebpack（または新しくTurbopack）ベースです。vinextはViteを採用することで、既存のViteエコシステムとの互換性を持ちつつ、ビルド速度の恩恵を受けています。

```typescript
// vinextのプロジェクト構造（Next.js互換）
// app/
//   layout.tsx   ← Server Component
//   page.tsx     ← Route Handler
// vinext.config.ts  ← Viteベースの設定
```

### テストを品質ゲートとして機能させる

2,000件超のテストは単なる数字ではありません。AIが生成したコードの品質を担保するために、テストが品質ゲートとして機能しています。「AIに書かせる→テストで検証する」というサイクルを維持することで、高速開発と品質を両立しています。

## vinextの現状と限界

vinextは現時点で**実験的プロジェクト**であることを明記する必要があります：

- Next.jsの全機能を網羅しているわけではない
- 本番環境での使用は現時点では推奨されない
- Cloudflare Workers環境での動作に最適化されている面がある

ただし、これをProof of Conceptとして読む場合、示唆するものは大きいです。

## 開発者へのインプリケーション

### Framework開発のコストが劇的に下がる

従来、フレームワーク開発は専門チームが何年もかけて取り組むものでした。vinextの事例は、AIの補助があれば**1人のエンジニアが特定用途のフレームワークを短期間で構築できる**ことを示しています。

汎用フレームワークを作る必要はありません。自社の特定ユースケースに特化した薄いフレームワーク層を作る、という選択肢が現実的になっています。

### テストが「AI開発の品質ゲート」になる

vinextの開発で注目すべきは、テストの扱い方です。AIが書いたコードに対して人間がテストを設計・管理し、品質の担保を自動化する。これはAI時代のTDDの新しい形と言えます。

### Viteエコシステムへの知識投資

Next.jsがwebpack→Turbopackという移行を進める一方、ViteベースのフレームワークはSvelteKit、Nuxt、Astroなど多数あります。Viteへの知識投資は、より広いフレームワークエコシステムで応用できます。

## まとめ

vinextはまだ実験段階のプロジェクトですが、「AIを使えばFramework開発のコストが劇的に下がる」という事実を証明しました。

コスト・品質・スピードを三立するためのノウハウとして、この方向性は引き続き追っていきます。
