2026年4月2日、Microsoft AIはMAIファミリーの新しいFoundational Modelとして、MAI-Transcribe-1、MAI-Voice-1、MAI-Image-2の3つを発表しました。このリリースが注目される理由はタイミングではなく、各モデルの技術的な具体性にあります。既存の代替手段に対する明確なPerformance上の優位性を打ち出しています。

本稿では各モデルの機能、性能特性、そしてMicrosoft Foundry経由でApplicationへ組み込む際の考え方を整理します。

## MAI-Transcribe-1：音声認識

MAI-Transcribe-1は25言語に対応した音声認識（Speech-to-Text）Modelです。主なメトリクスは以下の通りです：

- MicrosoftのAzure Fast提供比で**2.5倍高速**
- FLEURSベンチマークで上位25言語を通じた**平均WER（Word Error Rate）3.8%**
- OpenAIのWhisper-large-v3を全25言語で上回る
- クリーンなスタジオ録音だけでなく、実環境のノイズへの対応を設計段階から考慮

WhisperやAzure Speech-to-Textを現在使用しているApplicationにとって、直接的なBenchmarkの価値があります。WERの改善は小さくありません。実環境音声でのWERが5〜6%から3.8%に下がることは、顧客コール、会議録音、音声Interfaceでの転写誤りが実質的に減ることを意味します。

Pricing: 転写音声1時間あたり$0.36。

```python
# MAI-Transcribe-1でAudio転写する例 (Azure AI Foundry経由)
from azure.ai.inference import ChatCompletionsClient
from azure.core.credentials import AzureKeyCredential
import base64

client = ChatCompletionsClient(
    endpoint="https://YOUR_FOUNDRY_ENDPOINT.services.ai.azure.com/models",
    credential=AzureKeyCredential("YOUR_KEY"),
)

with open("meeting_audio.mp3", "rb") as f:
    audio_b64 = base64.b64encode(f.read()).decode()

response = client.complete(
    model="MAI-Transcribe-1",
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "audio_url", "audio_url": {"url": f"data:audio/mpeg;base64,{audio_b64}"}},
                {"type": "text", "text": "この音声を正確に文字起こししてください。"}
            ]
        }
    ]
)

print(response.choices[0].message.content)
```

## MAI-Voice-1：テキスト音声合成

MAI-Voice-1はTTS（Text-to-Speech）Modelで、現行TTS Serviceと差別化できる2つの能力を持ちます：

- **60秒分の自然な音声を1秒以内に生成** — Real-time factorが大幅に1.0を下回る
- **数秒の参照音声からCustom Voice作成**に対応
- 長文コンテンツ全体を通じてSpeaker Identityを維持

リアルタイムの生成速度はInteractiveなApplicationで重要です。音声AssistantやIVRシステムにおいて、TTSのLatencyがしばしばBottleneckになります。10秒分の音声合成に2〜3秒かかれば会話が不自然に感じられます。MAI-Voice-1のThroughputはその制約を排除します。

少量の参照音声からのCustom Voice Cloningも注目すべき機能です。これまで特定の話者音声を再現するには数時間分の録音セッションが必要でした。数秒の参照音声で済むなら、製品向けAI音声Personaの構築やAccessibility応用の可能性が大きく広がります。

Pricing: テキスト100万文字あたり$22。

```python
# MAI-Voice-1でCustom Voiceを使った音声生成
import requests

payload = {
    "model": "MAI-Voice-1",
    "input": "第4四半期の業績報告を始めます。主要メトリクスから確認しましょう。",
    "voice": "custom",
    "voice_reference_url": "https://your-storage.blob.core.windows.net/voices/speaker_reference.wav",
    "response_format": "mp3"
}

headers = {
    "Authorization": "Bearer YOUR_FOUNDRY_KEY",
    "Content-Type": "application/json"
}

response = requests.post(
    "https://YOUR_FOUNDRY_ENDPOINT.services.ai.azure.com/v1/audio/speech",
    json=payload,
    headers=headers
)

with open("output.mp3", "wb") as f:
    f.write(response.content)
```

## MAI-Image-2：画像生成

MAI-Image-2は画像生成Modelです：

- Arena.aiの画像生成**Leaderboardでトップ3入り**
- Foundry・Copilot上での生成速度が前世代比で**2倍高速化**
- Text-to-ImageおよびImage-to-Imageの両ワークフローに対応

Pricing: テキスト入力100万Tokenあたり$5、画像出力100万Tokenあたり$33。

Arena.aiのLeaderboardでのPositioningはMidjourneyやStable Diffusion 3と並ぶ品質を示しており、別の画像生成API依存を避けたいAzureユーザーにとって現実的な選択肢となります。

## Application Architectureへの組み込み

3つのModelを合わせると、コアとなるMultimodal Pipelineがカバーされます。「音声入力→テキスト→音声出力」、さらに画像生成を並列ブランチとして持つ構成です。3つすべてを必要とするApplication（視覚コンテンツも生成できる音声対応Assistant等）であれば、単一のAzure AI Foundry Endpointに集約することでAuth、課金、Rate Limit管理が簡素化されます。

より現実的な短期的活用ケースは選択的な採用です。現在サードパーティの転写Service、別のTTS API、別の画像生成Modelを使っているチームは、Foundryに集約することでExternal API依存を減らし、スケールでのトータルコスト削減が見込めます。

## 押さえておくべき背景

Microsoft MAI Modelは、Mustafa Suleyman氏が率いるMAI Superintelligenceチームが構築・保守しています。これは既存のAzure Cognitive ServicesやAzure OpenAIチームとは別組織で、OpenAIのModelをWrapするのではなく、競合する第一者Modelを開発しています。OpenAIへの単一ベンダー依存を懸念する組織にとって、明確なBenchmark比較を持つMicrosoft独自のAlternativeとして評価に値します。

FoundryへのIntegrationは標準的なAzure SDK Patternを通じて行われ、既存のAzure InfrastructureのIdentity、Network、Monitoring設定をそのまま活用できます。
