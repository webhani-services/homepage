Next.js 16.2 が 2026年3月18日にリリースされました。今回のリリースでは、日常的な開発体験を直接改善する実用的な機能が多く含まれています。特に注目すべきは、開発サーバーの大幅な高速化と、AI Agent を意識したツール連携の強化です。

## 開発サーバー起動が約400%高速化

最も体感しやすい変更点として、`next dev` の起動速度が前バージョン比で約400%高速化されています。大規模なプロジェクトでは起動に数十秒かかることもありましたが、16.2 ではその待ち時間が大幅に削減されます。

Turbopack をベースにしたこの改善は、Server Fast Refresh の導入と組み合わさることで、サーバー側コンポーネントの変更も細粒度でホットリロードできるようになっています。

```bash
# 従来の起動時間の目安 (16.1)
$ time next dev
# real: ~8s (中規模プロジェクト)

# 16.2 以降
$ time next dev
# real: ~2s (同等のプロジェクト)
```

レンダリング速度も約50%改善されており、CI/CD パイプラインでのビルド時間削減にも寄与します。

## AI Agent 対応の create-next-app

`create-next-app` が AI 対応プロジェクトをデフォルトでスキャフォールドするようになりました。具体的には、AI ツールが利用しやすい構造と設定が初期状態から含まれています。

```bash
npx create-next-app@latest my-app
# → AI-ready な構成でプロジェクトが作成される
```

これは単なる雛形の変更ではなく、AI Agent がコードを読み書きしやすいプロジェクト構造（型定義の整備、ファイル分離の徹底など）を標準化する意図があります。

## Browser Log Forwarding

開発中のブラウザエラーをターミナルに転送する機能が追加されました。これにより、AI Agent がターミナルを通じてブラウザ側のエラーを即座に把握し、修正提案を行えるようになります。

従来の開発フローでは、ブラウザのコンソールとターミナルを行き来する必要がありました。Browser Log Forwarding はこの摩擦を解消し、特に AI Coding Assistant との連携において大きな効果を発揮します。

```
# ターミナル出力例
[browser] TypeError: Cannot read properties of undefined (reading 'map')
    at ProductList (src/components/ProductList.tsx:24:18)
```

エラーの発生箇所がターミナルに表示されるため、Claude Code などのツールが自動的にファイルを特定して修正を試みることができます。

## Experimental Agent DevTools

実験的機能として、AI Agent がターミナル経由で React DevTools と Next.js の診断情報にアクセスできる Agent DevTools が追加されました。この機能を有効にすることで、Agent はコンポーネントツリーの状態やパフォーマンスのボトルネックを直接確認しながら作業を進めることができます。本番環境では使用せず、開発時のデバッグ支援として活用するのが適切です。

```js
// next.config.ts
const nextConfig = {
  experimental: {
    agentDevTools: true,
  },
};

export default nextConfig;
```

## Turbopack のその他の改善点

Turbopack に関しては以下の改善が含まれています：

- **Web Worker Origin**: WASM ライブラリの Worker 内での利用をより広範にサポート
- **Subresource Integrity**: JavaScript ファイルへの整合性チェックの追加
- **Tree Shaking of Dynamic Imports**: `dynamic import()` から未使用エクスポートを除去し、バンドルサイズを削減

特にサードパーティ製の WASM ライブラリを Web Worker で使用しているプロジェクトでは、Web Worker Origin の改善が直接的な恩恵をもたらします。

## webhani としての所見

今回のリリースで最も注目しているのは、AI Agent を開発ツールチェーンの一員として明示的に位置づけた点です。Browser Log Forwarding や Agent DevTools は、人間がブラウザとターミナルを往復して確認していた情報を、自動的に集約・転送する仕組みです。

これは「AI を使って開発する」という段階から「AI と共同で開発する」環境を整備するという方向性の表れです。webhani でも、社内プロジェクトや顧客向け開発において 16.2 へのアップグレードを積極的に進めていく予定です。

アップグレード自体は `package.json` の `next` バージョンを更新するだけで完了します：

```bash
npm install next@latest
```

ただし、`proxy.ts` の位置や `cacheHandlers`（複数形）への変更など、16.x 系の破壊的変更点は事前に確認しておくことを推奨します。

## まとめ

Next.js 16.2 は地味に見えて、実際の開発体験を着実に改善するリリースです。特に以下の3点は早めに試す価値があります：

- `next dev` の起動速度改善による反復開発の効率化
- Browser Log Forwarding による AI ツールとの連携強化
- Turbopack の Tree Shaking 強化によるバンドルサイズ削減

公式リリースノートは [nextjs.org/blog](https://nextjs.org/blog) で確認できます。
