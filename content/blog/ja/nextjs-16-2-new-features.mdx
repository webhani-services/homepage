# Next.js 16.2：AI対応 create-next-app と Turbopack Server Fast Refresh

Next.js 16.2が公開され、AIツールとの連携強化、開発速度の向上、セキュリティ面での改善が含まれています。実際の開発に影響する点を整理します。

## AI-ready create-next-app

`create-next-app` がAI Code GeneratorやAgent向けに最適化されました。

```bash
npx create-next-app@latest my-app
```

主な変更点は以下の通りです:

- **Browser Log Forwarding**: ブラウザのコンソールログがターミナルに転送されるようになり、AIエージェントがブラウザの状態をリアルタイムで確認できます
- **Dev Server Lock File Handling**: 開発サーバーの状態管理が改善され、複数のAgentが同じプロジェクトを操作する際の競合が防止されます
- **Experimental Agent DevTools**: まだ実験的ですが、AIエージェントがアプリの状態を検査・操作するためのDevToolsが追加されています

従来のCLIツールは人間の操作を前提に設計されていましたが、AIエージェントが主体のワークフローに対応した設計変更の方向性が見えます。

## Turbopack Server Fast Refresh

Turbopackにサーバーサイドの細粒度Hot Reloadが追加されました。

これまで Server Components に変更を加えると、Next.jsはページ全体の再描画を行っていました。Server Fast Refreshでは変更されたServer Componentのみを再レンダリングし、ページの状態を維持します。

開発中にフォームの入力内容やスクロール位置がコード変更のたびにリセットされる問題が軽減されます。特にデータ取得を伴う複雑なページでの開発体験が向上します。

## Web Worker Origin と WASM サポート

TurbopackのWeb Worker OriginによってWASMライブラリのWorker対応が広がりました。

```typescript
// WASM moduleをWorkerで使用する例
const worker = new Worker(new URL('./heavy-computation.worker.ts', import.meta.url));
```

画像処理やデータ解析などCPU負荷の高い処理をメインスレッドから切り離したい場面で有用です。

## Subresource Integrity（SRI）

JavaScriptファイルにSubresource Integrityのサポートが追加されました。

```html
<!-- 自動生成されるSRI hash付きscriptタグ -->
<script
  src="/_next/static/chunks/main.js"
  integrity="sha256-abc123..."
  crossorigin="anonymous"
></script>
```

CDNやサードパーティのJSが改ざんされた場合、ブラウザがそのスクリプトの実行をブロックします。コンテンツセキュリティの要件が厳しい企業向けサイトに特に有用です。

## セキュリティ修正

16.2では複数の高・中・低深刻度の脆弱性修正も含まれています。既存プロジェクトは早めのアップグレードが推奨されます。

```bash
npm install next@latest
```

## まとめ

Next.js 16.2の変更点はAIエージェントとの協調開発、開発体験の向上、セキュリティ強化という3つの軸で整理できます。特にTurbopack Server Fast RefreshはServer Componentsを多用するプロジェクトでの日々の開発スピードに直結します。

セキュリティ修正も含まれているため、既存プロジェクトはアップグレードを検討してください。
