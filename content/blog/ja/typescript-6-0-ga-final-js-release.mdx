TypeScript 6.0 が 2026 年 3 月 23 日に正式リリースされました。このリリースは単なるバージョンアップではなく、TypeScript の歴史における一つの区切りです。**これが JavaScript で書かれた TypeScript コンパイラの最終版**となります。

## 2 つの TypeScript が並走している

現在、開発チームは 2 つの TypeScript トラックを意識する必要があります。

- **TypeScript 6.0**（2026 年 3 月 23 日 GA）：JavaScript 実装の最終版
- **TypeScript 7.0**（2026 年 1 月 15 日 GA）：Go ネイティブ実装「Project Corsa」

TypeScript 7.0 はすでに正式リリース済みです。6.0 はその後に GA を迎えていますが、これは「7.0 への移行準備を完成させるためのマイルストーン」として位置づけられています。

## TypeScript 7.0 のパフォーマンス

Go ネイティブ実装による速度向上は測定済みです。

| Codebase | TypeScript 6.x | TypeScript 7.0 | 改善率 |
|---------|---------------|---------------|--------|
| VSCode（150 万行） | 89 秒 | 8.74 秒 | **10.2x** |
| メモリ使用量 | — | — | **約 3 分の 1** |

CI 環境でのコンパイル時間短縮は、ビルドコストと開発者体験の両方に直結します。特にコードベースが大きいプロジェクトほど恩恵が大きくなります。

## TypeScript 6.0 の主な変更点

6.0 は「7.0 への橋渡し」であると同時に、いくつかの重要な変更が含まれています。

### デフォルト値の変更

```json
// tsconfig.json — 6.0 以降の新しいデフォルト
{
  "compilerOptions": {
    "strict": true,          // 以前は false がデフォルト
    "module": "esnext",      // 以前は commonjs がデフォルト
    "target": "ES2025"       // 以前は ES3 がデフォルト
  }
}
```

既存プロジェクトで `tsconfig.json` に明示的な値を設定していない場合、動作が変わる可能性があります。アップグレード前に現在の設定を確認してください。

### ES2025 lib の追加

```typescript
// Temporal API の型が組み込みで利用可能に
const now = Temporal.Now.plainDateTimeISO();
console.log(now.toString()); // "2026-03-29T..."

// Map の upsert メソッド（Stage 4 提案）
const map = new Map<string, number>();
map.getOrInsert("count", 0);
map.getOrInsertComputed("total", () => calculateTotal());
```

### サブパスインポートのサポート

```typescript
// package.json の imports フィールドで定義した #/ プレフィックスをサポート
import { something } from '#/utils';

// package.json
{
  "imports": {
    "#/utils": "./src/utils/index.ts"
  }
}
```

Node.js の `nodenext` / `bundler` モジュール解決でのサブパスインポートが正式サポートされます。

## 移行の優先順位

**すぐに対応すべきこと**

1. tsconfig.json のデフォルト変更影響チェック（特に `strict`、`module`、`target`）
2. TypeScript 6.0 へのアップグレードとコンパイルエラーの確認

```bash
npm install typescript@6 --save-dev
npx tsc --noEmit  # エラーを事前確認
```

**中期的に検討すること**

3. TypeScript 7.0 へのアップグレード評価（Go ネイティブ、10x 高速）
4. ARM64 最適化パッチ（7.0.1、2026 年 3 月予定）の確認

## まとめ

TypeScript 6.0 GA は JavaScript 実装の終点であり、Go ネイティブ時代の開始を象徴するリリースです。7.0 がすでに安定版として利用可能である以上、今後の新規プロジェクトでは 7.0 を選択する理由が明確にあります。

既存プロジェクトについては、6.0 で設定の見直しと移行準備を行ってから 7.0 に進むのが現実的なパスです。コンパイル時間が開発体験のボトルネックになっているチームは、7.0 の移行を優先して検討する価値があります。
