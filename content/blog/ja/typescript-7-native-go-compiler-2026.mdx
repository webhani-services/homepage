## TypeScript 7.0、RC公開 — Goで再実装された新しい基盤

2026年6月中旬、TypeScript 7.0がRelease Candidateの段階に到達しました。Microsoft公式の発表によると、安定版のGA（General Availability）リリースはRCから約1ヶ月後を想定しています。

これまでのTypeScript（6.0まで）はJavaScriptで実装されていましたが、7.0から全く新しい基盤が登場します。その最大の特徴はネイティブバイナリ化です。コンパイラをGoで再実装することで、型チェック速度が劇的に向上しました。

Microsoft報告によれば、型チェックのパフォーマンスは従来比で**約10倍**の高速化を実現。VS Codeのコードベース（約150万行のTypeScript）の例では、型チェック時間が78秒から7.5秒に短縮されています。この改善は、単なる計測値の向上ではなく、開発者の日常体験、そしてCI/CDのフィードバック速度に直結します。

## ネイティブバイナリの仕組み — 「Strada」プロジェクト

TypeScript 7.0のコンパイラ再実装は、内部的には「Strada」と呼ばれるプロジェクトです。Microsoftのエンジニアリングチームは、既存のJavaScript実装を構造をそのまま保ちながら、ファイル単位でGoへと移植していきました。

重要なのは、**型チェックの動作は完全に等価**という設計方針です。新しいGoベースの実装は、既存のTypeScript 6.0と全く同じ診断結果を出力するように検証されています。チーム報告では、実装量が既存コードベースの約85%に到達した時点で、RCリリースが決定されました。

高速化の源泉は、ネイティブバイナリ実行とマルチスレッド並列処理（共有メモリの利用）の両立にあります。JavaScriptエコシステムの制約を脱することで、システムリソースをより効率的に活用できるようになったわけです。

## 本番環境への導入は段階的に — RCの扱い方

TypeScript 7.0 RCを試すなら、標準のnpmパッケージからインストールできます。

```bash
npm install -D typescript@rc
```

このコマンドで入手する`tsc`バイナリが、Goで実装されたネイティブコンパイラです（nightly版の`@typescript/native-preview`との違いについては後述）。

ただし、**RC段階であることを忘れずに**。本番のCI/CDパイプラインをいきなりRCに切り替えるのは避けるべきです。代わりに、**並行実行テスト**を推奨します。

### Shadow CI での検証例

既存の型チェックと並行して、RC版を実行する`package.json`スクリプトの例：

```json
{
  "scripts": {
    "type-check": "tsc --noEmit",
    "type-check:rc": "npm install -D typescript@rc && tsc --noEmit"
  }
}
```

CI環境では、通常の`type-check`と`type-check:rc`を別々のジョブとして実行し、診断結果（エラー数、警告の内容）を比較します。全く同じ結果が得られれば、切り替えの準備が整ったサインです。

```yaml
# GitHub Actions 例
jobs:
  type-check-stable:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm install
      - run: npm run type-check
  
  type-check-rc:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm install
      - run: npm run type-check:rc
```

### nightly版「native-preview」との使い分け

開発チームの先端への関心が高い場合、nightly版の`@typescript/native-preview`を使用することもできます。

```bash
npm install -D @typescript/native-preview
```

このパッケージでは、`tsgo`コマンドが有効になります。最新の実装試行版を試したいなら、こちらを選択してください。ただし、安定性はRC版より低いため、個人開発環境や実験的なプロジェクトに限定することをお勧めします。

## 型チェック高速化が生む実務上の変化

「10倍高速」という数字だけ聞くと、計測値の向上に思えるかもしれません。しかし実務規模では、このインパクトは極めて大きいです。

**エディタの応答性**が向上します。VS Codeでコードを入力するたびに、バックグラウンドで型チェックが走ります。これが7.5秒で終わるなら、ユーザーの入力ラグや補完候補の遅延が大幅に減ります。

**CI/CDのフィードバック速度が劇的に改善**します。大規模モノレポでは、型チェックがCI実行時間の長い部分を占めることが多いです。10倍の高速化は、パイプライン全体の実行時間を数分短縮できる可能性があります。

**エンジニアのイテレーション速度が上がります**。型チェックが足かせでなくなれば、開発者はコード品質の向上に集中できます。プリコミットチェックの実行が軽くなれば、チェックを厳しくすることも容易になります。

## 留意点 — プラットフォーム別バイナリと環境構築

TypeScript 7.0がネイティブバイナリ化されたということは、**プラットフォーム別にバイナリが用意される**ということです。

Windows、macOS（Intel/Apple Silicon）、Linux（x86_64/ARM）など、ターゲット環境に応じたバイナリがnpmパッケージに含まれます。これは通常、大きな問題ではありませんが、CI環境のランナーアーキテクチャが珍しい場合（例：ARMベースのカスタムランナー）、バイナリが提供されていないという状況が生じる可能性があります。

導入前に、**CI環境で実際に動作するか確認**することは必須です。Shadow CIテストの際に、正常に実行できるかを必ず検証してください。
