## AI가 AI를 만드는 시대

Anthropic이 공개한 수치 하나가 개발자 커뮤니티에서 주목받고 있습니다. Claude Code 코드베이스의 약 90%가 Claude Code 자체에 의해 생성되고 있다는 것입니다.

단순한 마케팅 문구가 아닙니다. Agentic Coding이 실험적 개념에서 실제 프로덕션 워크플로우로 전환되고 있다는 구체적인 신호입니다.

## Agentic Coding이란 무엇인가

AI 보조(AI-assisted) 코딩과 Agentic Coding의 차이는 본질적입니다.

초기 Copilot과 같은 도구는 개발자가 입력 중인 줄의 다음을 제안하는 방식이었습니다. 코드의 주도권은 개발자에게 있었고, AI는 입력 보조 수단이었습니다.

Agentic Coding에서는 AI가 작업을 받아 코드베이스를 분석하고, 계획을 수립하고, 구현하고, 테스트를 실행하고, 실패 시 원인을 파악해 수정합니다. 개발자는 이 루프 바깥에서 "무엇을 만들지"를 지시하고, "어떻게 만들지"의 대부분을 AI에 위임합니다.

```bash
# Claude Code에 작업을 위임하는 예시
claude "인증 미들웨어에 JWT 리프레시 토큰 로테이션 기능을 추가해주세요.
- Access Token 유효기간: 15분
- Refresh Token 유효기간: 7일
- ORM: Prisma + PostgreSQL
- 기존 테스트 스위트에 통합할 것"
```

모델은 기존 미들웨어를 읽고, 테스트 패턴을 파악한 뒤, 구현을 작성하고 테스트를 실행합니다. 개발자의 역할은 그 결과물을 검토하는 것으로 바뀝니다.

## "어렵고 직관에 반한다"는 현실

Anthropic 엔지니어들도 LLM을 활용한 프로그래밍이 "버튼 하나로 해결되는 마법"이 아니라고 인정합니다. 실제로 좋은 결과를 얻으려면 새로운 패턴을 익혀야 합니다.

실무에서 중요한 포인트를 정리했습니다.

**1. Context 품질이 결과물을 결정한다**

모호한 지시는 모호한 코드를 만듭니다. 프레임워크 버전, 기존 아키텍처 패턴, 제약 조건을 명시적으로 전달하면 기존 코드와 잘 맞는 구현이 나옵니다.

```markdown
## 항상 포함해야 할 Context
- Framework: Next.js 16 App Router (Pages Router 아님)
- 인증: Clerk, proxy.ts에 미들웨어 존재
- DB: Neon Serverless Postgres (Prisma)
- 실행 환경: Edge Runtime (Node.js API 사용 불가)

## 작업
인증되지 않은 사용자는 /sign-in으로,
구독이 없는 사용자는 /pricing으로 리다이렉트하는
미들웨어를 기존 proxy.ts에 통합해주세요.
새 파일을 만들지 말 것.
```

**2. 비판적 검토는 여전히 개발자의 몫**

AI가 생성한 코드도 미리뷰 없이 병합하는 코드와 같은 위험을 지닙니다. 보안 취약점, 성능 비효율, 기존 패턴과의 불일치를 찾아내는 능력은 여전히 개발자의 핵심 역량입니다.

**3. 짧은 루프가 긴 세션보다 낫다**

큰 기능을 한 번에 설명하는 것보다 작은 단계로 나누어 검증하면서 진행하는 방식이 더 좋은 결과를 냅니다. 구현 → 검토 → 수정 지시 → 재구현 사이클을 짧게 반복하는 것이 핵심입니다.

## 2026년 3월 기준 모델 현황

| 모델 | SWE-bench Verified | 특징 |
|---|---|---|
| Claude Opus 4.6 | ~80% | Agentic 작업에 최적화, 자율적 태스크 실행 |
| GPT-5.3-Codex | ~77% | Terminal-Bench 2.0 리드 |
| Llama 4 Scout | 진행 중 | 1000만 토큰 Context, 오픈소스 |

오픈소스 모델의 성장도 주목할 만합니다. Meta Llama 4 Scout는 1000만 토큰의 Context 윈도우를 제공하며, 코드 생성·문서 처리·분류 등 많은 작업에서 상용 모델과 견줄 만한 성능을 보이고 있습니다. 사내 인프라에서 추론을 완결하고 싶은 팀에게 현실적인 선택지가 됩니다.

## 엔지니어의 역할 재정의

Agentic Coding의 확산은 엔지니어를 불필요하게 만들지 않습니다. 일의 성격이 바뀝니다.

**아키텍처 설계 역량이 더 중요해진다**: AI가 로컬 구현을 담당할수록, 전체 시스템 구조를 올바르게 설계하는 책임은 엔지니어에게 남습니다. 잘 설계된 시스템은 AI에게 명확한 경계를 제공합니다.

**코드 읽기 능력이 쓰기 능력만큼 중요해진다**: AI가 초안을 작성하면 개발자는 검토에 더 많은 시간을 씁니다. 200줄의 TypeScript 코드에서 미묘한 문제를 빠르게 찾아내는 능력은 갈수록 가치가 높아집니다.

**프롬프트 설계는 하나의 기술이다**: 문제를 명확한 단위로 분해하고, 적절한 Context를 제공하고, 모델의 출력에 올바른 피드백을 주는 능력은 연습으로 향상됩니다.

**테스트를 먼저 작성하면 AI를 올바른 방향으로 유도할 수 있다**: TDD는 Agentic Coding과 잘 맞습니다. 실패하는 테스트를 먼저 작성하고, 이를 목표로 모델에 구현을 맡기면 명확한 완료 기준이 생깁니다.

## 정리

Anthropic의 90% 수치는 하나의 이정표입니다. 앞으로 AI가 더 많은 루틴 구현을 담당하는 방향으로 흐름이 이어질 것입니다. 새로운 워크플로우를 일찍 습득하고, 적절한 검토 습관을 갖추고, AI가 대체하기 어려운 역량을 날카롭게 유지하는 엔지니어가 향후 12~24개월 동안 뚜렷한 경쟁력을 갖게 될 것입니다.
