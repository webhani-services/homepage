2026년 4월 27일 AWS 주간 라운드업에서 Amazon Bedrock AgentCore CLI가 발표되었습니다. MCP(Model Context Protocol)를 기반으로 AI Agent가 외부 서비스와 연동하여 인프라 인시던트 조사부터 해결까지 전 과정을 자동화합니다.

## AgentCore CLI 개요

AgentCore는 Bedrock 위에서 동작하는 AI Agent 런타임 환경입니다. CLI를 통해 MCP 기반 Tool 연결을 활용하고, 모니터링 시스템에서 데이터를 가져오거나 Runbook을 실행하는 Agentic 워크플로우를 구성할 수 있습니다.

인시던트 자동화 흐름:

1. Alert 발생 → AgentCore Agent가 수신
2. Log, Metrics, Trace 자동 수집 및 상관관계 분석
3. 근본 원인 파악 및 해결 방안 제시
4. 온콜 엔지니어의 승인 후 조치 자동 실행

완전 자동화가 아니라 Agent가 데이터 수집과 분석을 담당하고, 중요한 결정은 사람이 내리는 구조입니다.

## EKS Auto Mode와의 조합

같은 주에 GA된 EKS Auto Mode는 VPC CNI 설정, Load Balancer 프로비저닝, DNS 관리를 자동화합니다.

```yaml
# EKS Auto Mode Node Pool 설정
apiVersion: eks.amazonaws.com/v1
kind: NodePool
metadata:
  name: general-purpose
spec:
  template:
    spec:
      nodeClassRef:
        name: default
      requirements:
        - key: "karpenter.sh/capacity-type"
          operator: In
          values: ["on-demand"]
        - key: "kubernetes.io/arch"
          operator: In
          values: ["amd64"]
  disruption:
    consolidationPolicy: WhenUnderutilized
    consolidateAfter: 30s
```

EKS Auto Mode가 인프라 레이어의 운영 부담을 줄이고, AgentCore가 인시던트 대응의 인지 부담을 처리합니다. 두 가지를 조합하면 엔지니어가 아키텍처 결정에 집중할 수 있는 환경이 만들어집니다.

## MCP의 역할

MCP(Model Context Protocol)는 Anthropic이 제안한 Agent-Tool 통신 표준 프로토콜입니다. AWS가 AgentCore에 MCP를 채택한 것은 이 프로토콜이 인프라 수준의 표준으로 자리잡고 있음을 보여줍니다.

```python
# Bedrock Agent 호출 예시
import boto3

agent_runtime = boto3.client("bedrock-agent-runtime", region_name="us-east-1")

response = agent_runtime.invoke_agent(
    agentId="your-agent-id",
    agentAliasId="TSTALIASID",
    sessionId="incident-session-001",
    inputText=(
        "프로덕션 API 레이턴시가 10분간 300% 급증했습니다. "
        "근본 원인을 분석하고 해결 방안을 제시해주세요."
    ),
)

for event in response.get("completion", []):
    if chunk := event.get("chunk"):
        print(chunk["bytes"].decode("utf-8"), end="")
```

## 도입 시 주의사항

Agent 기반 DevOps의 주요 위험은 잘못된 근본 원인 분석에 대한 과신입니다. 실용적인 대응 방법:

- **읽기 전용부터 시작**: 처음에는 관찰만 하는 Agent로 시작합니다
- **명시적 승인 게이트**: 모든 쓰기 작업은 사람의 승인을 필요로 합니다
- **감사 로깅**: Agent의 모든 행동과 판단을 추적 가능하게 합니다

webhani에서는 현재 초기 인시던트 트리아지에 AgentCore를 파일럿 적용하고 있으며, Agent가 증거를 제시하고 사람이 최종 결정을 내리는 구조를 유지하고 있습니다. 실제 인시던트 이력과 비교해 진단 정확도를 검증할 때까지 이 경계를 유지할 계획입니다.

## 정리

AgentCore CLI는 AWS가 Agentic DevOps 방향으로 내딛은 가장 구체적인 한 걸음입니다. 읽기 전용 트리아지 자동화부터 시작하고, 실제 인시던트 이력과 비교해 Agent 진단의 정확도를 검증한 후 점진적으로 자동화 범위를 넓히는 것을 권장합니다. 도구는 준비되어 있고, 검증 작업은 팀의 몫입니다.
