## 소개

웹 애플리케이션에서 이메일 전송은 필수적인 기능 중 하나입니다. 이 글에서는 Next.js App Router와 AWS SES(Simple Email Service)를 조합하여 이메일 전송을 구현하는 방법을 소개합니다.

## AWS SES란?

Amazon Simple Email Service(SES)는 AWS가 제공하는 클라우드 기반 이메일 전송 서비스입니다. 높은 전송 성능과 비용 효율성을 겸비하고 있습니다.

### 주요 특징

- **높은 도달률**: AWS 인프라를 활용한 신뢰성 높은 이메일 전송
- **비용 효율**: 전송량에 따른 종량 과금제
- **유연한 API**: REST API와 SMTP 인터페이스 모두 지원

## 구현 절차

### 1. 패키지 설치

```bash
npm install @aws-sdk/client-ses
```

### 2. SES 클라이언트 설정

```typescript
import { SESClient, SendEmailCommand } from "@aws-sdk/client-ses";

const sesClient = new SESClient({
  region: process.env.AWS_REGION,
  credentials: {
    accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
    secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
  },
});
```

### 3. API Route 생성

Next.js App Router의 Route Handler를 사용하여 이메일 전송 엔드포인트를 생성합니다.

```typescript
export async function POST(request: Request) {
  const { name, email, message } = await request.json();

  const command = new SendEmailCommand({
    Source: "noreply@example.com",
    Destination: {
      ToAddresses: ["admin@example.com"],
    },
    Message: {
      Subject: { Data: `문의: ${name}` },
      Body: {
        Text: { Data: `이름: ${name}\n이메일: ${email}\n내용: ${message}` },
      },
    },
  });

  await sesClient.send(command);
}
```

## 마무리

AWS SES와 Next.js를 조합하면 심플하면서도 신뢰성 높은 이메일 전송 기능을 구현할 수 있습니다. 프로덕션 환경에서는 Rate Limiting과 에러 처리도 적절히 설정하세요.
