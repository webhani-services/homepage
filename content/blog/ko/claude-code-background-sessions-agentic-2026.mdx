## 백그라운드 세션이란

Claude Code에 `/bg` 커맨드가 추가되었습니다. 현재 활성 세션을 Agent View라는 백그라운드 처리 모드로 이동하는 기능으로, Claude가 독립적으로 작업을 계속 진행하는 동안 새 세션을 열거나 다른 일을 할 수 있습니다.

실용적 의미는 이렇습니다. 대규모 리팩터링, 테스트 실패 분석, 빌드 모니터링처럼 시간이 걸리는 작업을 위해 터미널 앞에 대기할 필요가 없어집니다. 위임하고, 백그라운드로 보내고, 완료 후 결과를 확인하면 됩니다. AI 코딩 툴과의 인터랙션이 '대화형'에서 '비동기 위임형'으로 전환되는 것입니다.

## 기본 사용법

Claude Code 터미널 세션에서 작업을 지시한 후:

```
/bg
```

현재 작업이 Agent View로 이동합니다. 이후 가능한 것:
- **새 세션 시작** — 다른 작업 진행
- **실시간 진행 상황 확인** — Agent View의 인라인 업데이트
- **터미널 종료 후 재연결** — Remote Control 기능으로 나중에 접속

Agent View는 여러 백그라운드 작업을 병렬로 표시하며, Claude가 진행하는 상황을 인라인 업데이트로 보여줍니다.

## 실용적인 워크플로 패턴

### 패턴 1: 대규모 리팩터링 위임

TypeScript strict 모드 마이그레이션, 의존성 업그레이드, 코드베이스 전체 리네임 — 시간은 걸리지만 규칙이 명확한 작업이 적합합니다.

```
# 범위와 제약이 명확한 지시 예시
> tsconfig.json에서 strict: true를 활성화하고 src/ 전체의 타입 에러를 수정해주세요.
  다음 제약을 지켜주세요:
  - 런타임 로직은 변경하지 마세요 — 타입 수정만 해주세요
  - any 타입 사용은 금지
  - 함수 시그니처 변경이 필요한 경우 최소한의 변경만 해주세요
  완료 후 npm run type-check를 실행해서 에러가 0인지 확인해주세요.

/bg  # 백그라운드로 이동, 새 세션 열기
```

Claude가 타입 에러를 처리하는 동안 다른 작업을 진행할 수 있습니다.

### 패턴 2: 테스트 실패 조사

```
> npm test 출력에서 실패하는 유닛 테스트를 조사하고 수정해주세요.
  E2E 테스트는 제외해주세요. 테스트 assertions는 변경하지 마세요 —
  assertions가 잘못된 경우 삭제하지 말고 주석으로 표시해주세요.
  완료 후 npm test를 다시 실행해서 유닛 테스트가 모두 통과하는지 확인해주세요.

/bg
```

### 패턴 3: 빌드 및 배포 모니터링

```
> docker compose build를 실행해주세요. 성공하면 docker compose up -d를 실행하고
  모든 컨테이너가 헬스 체크를 통과하는지 확인해주세요.
  실패하면 정확한 에러 출력을 보고하고 중단해주세요.

/bg  # Claude가 결과에 따라 액션 수행
```

## 백그라운드 작업을 위한 효과적인 지시 작성

백그라운드 실행의 품질은 지시의 구체성에 크게 의존합니다. Claude가 후속 질문을 할 수 없을 때는 판단을 스스로 내려야 하므로, 지시가 구체적일수록 그 판단의 정확도가 높아집니다.

```
# 백그라운드에 부적합한 모호한 지시
> 타입 에러 수정해주세요

# 자율 실행에 적합한 구체적인 지시
> src/components/에서만 TypeScript 타입 에러를 수정해주세요.
  제약 조건:
  - 테스트 파일은 수정하지 마세요
  - any 타입은 사용하지 마세요
  - 함수 시그니처 변경이 필요한 경우 최소한의 변경만 해주세요
  모든 수정 후 npx tsc --noEmit를 실행해서 에러가 없는지 확인해주세요
```

명확히 해야 할 요소는 **범위**(어떤 파일), **제약 조건**(하지 말아야 할 것), **성공 기준**(검증 방법), **불확실한 상황의 행동 방침** 4가지입니다.

## Dispatch와 Remote Control

두 가지 추가 기능이 백그라운드 세션을 터미널 이상으로 확장합니다.

**Dispatch**는 CI/CD 스크립트, 외부 툴링, 스케줄된 잡에서 프로그래밍 방식으로 작업을 제출할 수 있게 합니다. Claude Code가 작업을 받아 실행하고 결과를 반환합니다. 실용적인 예: 새 Pull Request가 생성될 때마다 자동으로 Claude Code 리뷰 세션을 트리거하는 방식입니다.

**Remote Control**은 터미널 외부에서 실행 중인 Claude Code 세션에 연결하는 기능입니다. 작업을 시작하고 노트북을 닫은 뒤, 몇 시간 후 같은 세션으로 재연결할 수 있습니다. 기기 간 인계나 퇴근 전 컨텍스트 전환에 유용합니다.

## Chyros: 다음 단계

"Chyros"는 계획 중인 Claude Code 기능의 내부 코드명으로, 터미널 세션 없이도 지속적으로 실행되는 백그라운드 데몬을 목표로 합니다. 현재 `/bg` 커맨드가 활성 세션을 백그라운드로 보내는 방식이라면, Chyros는 설계상 독립적으로 비동기 실행하는 형태입니다.

아직 출시되지 않았지만 방향은 명확합니다. 언제나 실행 중인 에이전트에게 작업을 할당하고 자신의 일정에 맞춰 결과를 받는 형태입니다.

## 현재 제약 사항

- 백그라운드 세션은 최신 버전의 Claude Code가 필요합니다
- Dispatch는 아직 베타 단계로 모든 사용자에게 완전히 제공되지 않습니다
- 같은 파일을 수정하는 병렬 백그라운드 작업은 충돌이 발생할 수 있습니다. 여러 에이전트를 동시에 실행할 때는 별도 디렉터리로 범위를 분리하세요

## 마무리

`/bg` 커맨드는 시간이 걸리지만 규칙 기반인 작업에 가장 효과를 발휘합니다 — 리팩터링, 테스트 수정, 코드 마이그레이션, 빌드 모니터링. 제품 방향성에 대한 맥락적 판단이 필요한 작업은 계속 대화형 방식을 유지하는 것이 낫습니다.

명확하게 범위가 정해진, 잘 제약된 작업부터 시작하세요. 지시가 구체적일수록 자율적인 실행의 신뢰성이 높아집니다. 백그라운드 모드가 성숙해지면서 'AI 페어 프로그래머'와 '작업 큐를 처리하는 AI 팀원'의 경계가 좁아지고 있습니다.
