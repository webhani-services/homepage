## 무엇이 달라졌나

Claude Code에 리서치 프리뷰로 「Dynamic Workflows」라는 새로운 기능이 추가되었습니다. 기존 AI 코딩 도구는 파일 단위, 함수 단위의 작업을 하나씩 진행하는 페어 프로그래머 방식으로 동작했습니다. Dynamic Workflows는 그 전제를 근본적으로 바꿉니다.

단일 Session 안에서 수백 개의 병렬 Subagent를 실행하여 대규모 Codebase 전체에 걸친 작업을 Claude가 자율적으로 완수합니다.

---

## 동작 구조

Dynamic Workflows는 다음 네 단계로 나뉩니다.

**1. 계획 단계**  
Claude가 작업 전체를 분석하고 의존 관계를 파악한 뒤, 병렬 실행 가능한 서브태스크로 분해합니다. 몇 개의 Agent를 실행할지, 각 Agent가 무엇을 담당할지를 결정합니다.

**2. 병렬 실행 단계**  
수백 개의 Subagent가 동시에 실행되며 각자 담당 범위(파일, 모듈, 논리 단위)를 처리합니다. Agent 간 충돌은 Claude가 관리합니다.

**3. 검증 단계**  
기존 Test Suite를 실행하여 모든 변경 사항이 사양을 만족하는지 확인합니다. 테스트 통과 여부가 완료 기준이 됩니다.

**4. 인계 단계**  
검증이 완료된 변경 사항을 Diff 형태로 사용자에게 제시하고 최종 확인을 요청합니다.

핵심은 Claude가 단순히 지시를 따르는 것이 아니라 **Test Suite 통과를 스스로 완료 기준으로 삼는다**는 점입니다. 덕분에 사람이 매 단계를 확인하지 않아도 품질 기준이 유지됩니다.

---

## 주요 사용 사례: Codebase 스케일 Migration

Dynamic Workflows가 가장 큰 효과를 발휘하는 곳은 규칙적이지만 수작업으로는 방대한 작업입니다.

- 10만 줄 이상의 Node.js 프로젝트를 ESM으로 마이그레이션
- 오래된 ORM을 새로운 ORM으로 전체 전환
- Deprecated API 호출을 모든 파일에서 일괄 교체

이런 작업은 원칙적으로는 단순하지만, 대규모 Codebase에서 수동으로 진행하면 몇 주가 걸리고 실수가 발생하기 쉽습니다.

```bash
# Dynamic Workflows 활용 예시
> 이 프로젝트 전체를 CommonJS에서 ESM으로 마이그레이션해 주세요.
  기존 테스트는 모두 통과해야 합니다.

# Claude의 자율 처리 과정
[계획 수립]
  200개 파일 변경 필요 확인
  14개 순환 의존성 매핑
  __dirname 대체 전략 결정: import.meta.dirname

[200개 Subagent 병렬 실행]
  Agent-001: src/index.js ✓
  Agent-002: src/utils/logger.js ✓
  Agent-047: src/routes/api.js ✓ (순환 의존성 해결)
  ...

[Test Suite 실행]
  300/300 테스트 통과

[완료 보고]
  Migration 완료. Diff를 검토해 주세요.
```

## Test Suite가 핵심인 이유

Dynamic Workflows의 설계에서 중요한 것은 **Test Suite가 완료 기준**이 된다는 점입니다.

- 테스트를 갖춘 프로젝트일수록 Claude가 안전하게 대규모 변경을 적용할 수 있습니다
- Unit Test는 개별 컴포넌트를, Integration Test는 모듈 간 연동을 검증합니다
- 테스트 커버리지가 낮은 프로젝트에서는 Dynamic Workflows의 효과가 제한적입니다

Test Suite는 이제 인간을 위한 회귀 방지 장치일 뿐만 아니라, **AI가 자율적으로 동작하기 위한 인터페이스**이기도 합니다.

## 기존 AI 코딩과의 비교

| 관점 | 기존 AI 코딩 | Dynamic Workflows |
|------|------------|------------------|
| 작업 단위 | 파일 / 함수 | Codebase 전체 |
| 병렬성 | 없음 | 수백 개의 Subagent |
| 완료 판단 | 사람이 매 단계 확인 | Test Suite 자동 검증 |
| 사람의 역할 | 순차 리뷰 | 사양 정의 + 최종 확인 |
| 적합한 작업 | 기능 추가, 버그 수정 | Migration, 일괄 변환 |

## 실전 준비 사항

Dynamic Workflows를 최대한 활용하려면 프로젝트 측의 준비가 중요합니다.

**테스트 커버리지 높이기**  
테스트가 AI의 판단 기준이 됩니다. Unit Test뿐 아니라 Integration Test도 정비해 두면 복잡한 Migration에도 대응할 수 있습니다.

**테스트를 사양으로 작성하기**  
테스트가 구현 세부 사항이 아니라 의도를 명확히 표현하도록 작성하면, Claude의 계획 단계 정확도가 높아집니다.

**소규모부터 시작하기**  
처음부터 프로젝트 전체를 대상으로 하지 말고, 단일 모듈이나 서브시스템부터 Dynamic Workflows를 적용해 동작 방식을 파악하세요.

## 정리

- Dynamic Workflows는 Claude Code의 리서치 프리뷰 기능
- 수백 개의 병렬 Subagent로 Codebase 스케일 작업을 자율 처리

AI 코딩이 페어 프로그래머에서 엔지니어링 팀으로 진화하고 있습니다. 그 변화에 얼마나 준비되어 있느냐가 개발 팀의 생산성을 결정짓는 시대가 되었습니다.
