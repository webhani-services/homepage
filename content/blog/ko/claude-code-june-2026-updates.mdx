# Claude Code 2026년 6월 업데이트: Fallback Model, Skill 자동 로드, 보안 강화

Claude Code가 2026년 6월에 여러 실용적인 업데이트를 받았습니다. 개발 워크플로우에 직접적으로 영향을 미치는 주요 변경사항을 살펴봅니다.

## Fallback Models 설정

새로운 `fallbackModel` 설定으로 기본 모델이 과부하 상태이거나 사용 불가할 때 순서대로 시도할 모델을 최대 3개까지 지정할 수 있습니다.

```json
{
  "model": "claude-opus-4-8",
  "fallbackModel": [
    "claude-sonnet-4-6",
    "claude-haiku-4-5"
  ]
}
```

장시간 실행되는 Agent 태스크나 배치 처리 Pipeline에서 Rate Limit에 도달하면 기존에는 작업이 중단되었습니다. Fallback Model 설정을 통해 Claude Code가 자동으로 대체 모델로 전환하여 작업을 이어갑니다.

## Skill 디렉토리 자동 로드

`.claude/skills` 디렉토리의 Plugin이 Marketplace 설치 없이 자동으로 로드됩니다. 재시작 없이 Skill 디렉토리를 다시 스캔하는 `/reload-skills` 명령어도 추가되었습니다.

팀에서 공유 Repository로 Custom Skill을 관리하는 경우, `git pull` 만으로 최신 Skill이 즉시 반영됩니다.

## Deny Rule의 Glob 패턴 지원

Deny Rule의 tool-name 위치에서 glob 패턴을 사용할 수 있게 되었습니다. `"*"` 를 지정하면 모든 Tool을 거부합니다.

```json
{
  "permissions": {
    "deny": [
      "Bash(*)",
      "Write(public/**)"
    ]
  }
}
```

특정 디렉토리 쓰기나 Shell 실행을 제한하면서 코드 읽기와 분석은 허용하고 싶은 팀 환경에서 활용할 수 있습니다.

## Cross-Session 보안 강화

다른 Claude Session에서 `SendMessage`로 중계된 메시지는 더 이상 User 권한을 갖지 않습니다. Multi-Agent 구성에서 Subagent가 다른 Session을 통해 권한을 에스컬레이션하는 위험이 줄어듭니다.

## MessageDisplay Hook

새로운 `MessageDisplay` Hook 이벤트로 Assistant 메시지 텍스트를 표시 전에 변환하거나 숨길 수 있습니다. 출력 필터링이나 팀용 Custom UI 구축에 활용할 수 있습니다.

## Vim Mode 개선

Vim 모드의 NORMAL 모드에서 `/` 키가 이제 역방향 히스토리 검색을 열어 Bash와 Zsh vi-mode와 동일한 동작을 합니다.

## 정리

이번 업데이트는 굵직한 기능보다는 실제 개발 현장의 불편함을 해소하는 개선들입니다. Fallback Models는 CI Pipeline과 Agent 워크플로우의 안정성을 높이고, Skill 자동 로드는 팀 온보딩을 단순화합니다. Multi-Agent 또는 공유 환경에서 Claude Code를 운영한다면 보안 강화도 주목할 만합니다.

`.claude/settings.json`을 점검하고 Fallback Models 설정을 적용해보시기 바랍니다.
