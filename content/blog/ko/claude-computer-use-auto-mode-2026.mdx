2026년 3월 24일, Anthropic이 Claude의 주요 업데이트를 발표했습니다. Pro·Max 플랜 사용자를 대상으로 Claude Code와 Cowork에서 **Computer Use** 기능이 제공되고, **Auto Mode** Research Preview도 시작되었습니다.

## Computer Use란 무엇인가

Computer Use는 Claude가 화면을 보고 직접 조작할 수 있는 기능입니다. 파일 열기, 터미널 명령 실행, 브라우저 네비게이션, UI 요소 클릭 등을 수행할 수 있습니다.

기존의 Tool Calling은 모든 액션을 구조화된 API 호출로 감싸야 했지만, Computer Use를 통해 Claude는 API를 제공하지 않는 GUI 애플리케이션과도 직접 상호작용할 수 있습니다.

```bash
# 멀티스텝 디버깅 태스크 위임 예시
# 지시: "Integration Test가 실패하는 원인을 찾아서 수정해주세요"
# Claude가 자율적으로 실행:
# 1. 터미널에서 테스트 Suite 실행
# 2. 실패 Output 읽기
# 3. 관련 소스 파일 열기
# 4. 수정 적용
# 5. 테스트 재실행 및 결과 확인
```

"지시 → Claude 제안 → 사람이 실행"이라는 사이클이, 일부 태스크에서는 "지시 → Claude 실행"으로 단축됩니다.

## Auto Mode: 제어된 자율성

Auto Mode는 Claude가 "어떤 액션을 자율적으로 실행해도 되는지" 스스로 판단하는 모드입니다. 현재는 Research Preview 단계로 허용 범위가 제한되어 있습니다. 파일 삭제, 외부 API 호출, 프로덕션 배포 등 되돌리기 어려운 액션은 현재 자동 실행이 차단되거나 확인이 필요합니다.

Anthropic의 신중한 접근은 타당합니다. 잘못된 판단으로 취소 불가능한 액션을 실행하는 Agent는 없는 것보다 못합니다.

Auto Mode의 현재 범위에 적합한 활용 사례:

- Lint 실행 및 자동 수정 적용
- 테스트 Suite 실행 및 실패 요약
- 리팩토링 전 관련 파일 탐색
- 코드 변경에 따른 주석·문서 자동 업데이트

## Claude Sonnet 4.6 성능 향상

이번 업데이트에서는 코딩, 장문 컨텍스트 추론, Agent 플래닝, 디자인 작업 등 여러 영역에서 성능이 향상되었습니다. 베타로 제공되는 1M 토큰 Context Window를 통해 대규모 Codebase 전체를 한 번에 분석하는 워크플로우가 실용적인 수준이 되었습니다.

## 도입 전 체크포인트

**액션 범위를 명확히 정의하세요.** 명시적인 경계 없이는 Agent가 의도하지 않은 Directory를 탐색하거나 명령을 실행할 수 있습니다.

**프로덕션 접근 권한은 제한하세요.** Computer Use와 프로덕션 Credential의 조합은 보안 위험이 될 수 있습니다. Agent 권한은 최소 권한 원칙에 따라 설정하세요.

**Auto Mode는 아직 초기 단계입니다.** Research Preview이므로 동작이 변경될 수 있습니다. 프로덕션 크리티컬 자동화는 정식 출시 이후 도입을 권장합니다.

## 전체적인 흐름

Claude, GPT-5.4, Gemini 3.1 Pro에서 공통적으로 나타나는 트렌드는 "제안"에서 "실행"으로의 전환입니다. 개발자 입장에서는 제안을 검토하고 직접 실행하는 방식 대신, 태스크 범위를 정의하고 결과를 검증하는 방식으로 협업 모델이 바뀌고 있습니다.

되돌릴 수 있는 태스크부터 시작해 신뢰 범위를 점진적으로 넓혀가는 접근이 현실적입니다. 지금 AI 워크플로우를 위한 명확한 태스크 정의를 설계하는 팀이 이 변화에서 가장 큰 이점을 얻을 것입니다.
