2026년 4월, Anthropic은 Claude Mythos Preview를 공개했습니다. 이 특화된 AI 에이전트는 주요 운영체제와 웹 브라우저 전반에서 수천 개의 제로데이 취약점을 발견했습니다. AI 기반 보안 연구가 어디까지 왔는지를 보여주는 이 발표는, 모든 엔지니어링 팀이 주목해야 할 실질적인 함의를 담고 있습니다.

## Claude Mythos는 무엇인가

Claude Mythos는 범용 모델이 아닙니다. Anthropic의 가장 강력한 에이전틱 추론 능력을 기반으로 구축되었지만, 취약점 발견, exploit 개발, 다단계 공격 체인 구성 등 오펜시브 보안 컨텍스트에 특화되어 있습니다.

영국 AI Safety Institute(AISI)의 평가에 따르면, Mythos는 32단계로 구성된 사이버 레인지를 처음부터 끝까지 스스로 해결했습니다. 각 단계마다 사람의 개입 없이 자율적으로 다음 행동을 결정하며 진행한 것입니다.

Anthropic이 시연한 가장 인상적인 사례는 브라우저 exploit 생성이었습니다. Mythos는 네 가지 별개의 취약점을 체이닝하여, 렌더러와 운영체제 샌드박스를 모두 탈출하는 복잡한 JIT 힙 스프레이를 포함한 exploit를 자율적으로 작성했습니다. 숙련된 보안 연구자가 수 주에 걸쳐 수행하는 작업을 AI가 독립적으로 완수한 것입니다.

## Project Glasswing

Anthropic은 Mythos를 일반에 공개하지 않습니다. 대신, Project Glasswing이라는 기술 기업 연합을 통해 통제된 방식으로만 접근권을 부여합니다.

현재 참여 기관으로는 AWS, Apple, Microsoft, Google, CrowdStrike, Palo Alto Networks 등이 포함되어 있으며, 약 40개 이상의 조직이 접근권을 받았습니다. 목표는 명확합니다. Mythos의 오펜시브 역량을 방어적 목적으로 전환하여, 공격자보다 먼저 취약점을 찾고 수정하는 것입니다.

Anthropic의 장기 목표는 적절한 안전 장치와 운영 제어가 갖춰진 이후, Mythos급 모델을 합법적인 사이버 보안 목적으로 대규모 배포하는 것입니다.

## 기업 보안팀에 미치는 영향

대부분의 엔지니어링 팀은 Mythos에 직접 접근할 수 없을 것입니다. 그렇다고 이 변화가 무관한 것은 아닙니다.

**위협 모델 재조정이 필요합니다**

고급 공격은 그동안 숙련된 공격자의 수가 제한적이라는 전제 위에 있었습니다. AI 기반 오펜시브 도구는 이 방정식을 바꿉니다. 이전에는 전문가가 수 주를 투자해야 가능했던 제로데이 체이닝이, 더 광범위한 위협 행위자들에게 접근 가능해질 수 있습니다. AI로 역량이 강화된 공격자를 위협 모델에 포함시켜야 합니다.

**방어 속도가 공격 속도를 따라가야 합니다**

연간 단위의 모의 침투 테스트와 분기별 보안 리뷰는 지속적인 AI 기반 취약점 발견 속도와 맞지 않습니다. 개발 워크플로우에 자동화된 지속적 보안 스캐닝을 통합해야 합니다.

## CI/CD에 지속적 보안 통합하기

실질적인 대응은 개발의 모든 단계에 자동화된 보안 점검을 계층화하는 것입니다. 다음은 시작점이 될 수 있는 설정입니다.

```yaml
# .github/workflows/security.yml
name: Security Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:

jobs:
  sast:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Semgrep SAST 실행
        uses: semgrep/semgrep-action@v1
        with:
          config: "p/owasp-top-ten p/javascript p/typescript"
        env:
          SEMGREP_APP_TOKEN: ${{ secrets.SEMGREP_APP_TOKEN }}

      - name: 의존성 취약점 검사
        run: npm audit --audit-level=high

      - name: 시크릿 탐지
        uses: trufflesecurity/trufflehog@main
        with:
          path: ./
          base: ${{ github.event.repository.default_branch }}

  container:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Docker 이미지 빌드
        run: docker build -t app:${{ github.sha }} .

      - name: Trivy 컨테이너 스캔
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: "app:${{ github.sha }}"
          severity: "CRITICAL,HIGH"
          format: sarif
          output: trivy.sarif

      - name: GitHub Security 탭에 업로드
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: trivy.sarif
```

Node.js나 Python 백엔드를 사용하는 팀이라면 PR 단계에서 CodeQL이나 Snyk를 추가하면, 패턴 매칭 SAST 도구가 놓치는 취약점 클래스를 잡을 수 있습니다.

## "Mythos급"이 시사하는 것

"Mythos급"이라는 표현은 특정 모델이 아닌 역량의 수준을 의미합니다. 취약점을 자율적으로 체이닝하고, 동작하는 exploit를 생성하며, 새로운 타깃에 적응하는 능력은 이제 실증된 현실입니다.

Project Glasswing에 참여하는 보안 벤더들(CrowdStrike, Palo Alto Networks)은 이 역량을 차세대 제품에 통합할 것입니다. 앞으로 12~18개월 안에 AI 기반 취약점 탐지가 중견 규모 팀도 접근 가능한 주류 보안 도구에 등장할 것으로 예상됩니다.

## 실질적인 우선순위

Project Glasswing 참여 기관 규모로 운영되지 않는 팀이 지금 당장 할 수 있는 일은 다음과 같습니다.

1. **보안을 왼쪽으로 이동**: IDE 레벨 SAST 플러그인과 pre-commit 훅을 추가하여 문제가 리뷰 큐에 들어오기 전에 잡기
2. **CI/CD 자동화**: 모든 PR에서 SAST, 의존성 스캔, 시크릿 탐지 실행
3. **지속적 위협 모델링**: 연간 모의 침투에서 지속적 자동화 스캐닝과 주기적 전문가 평가를 병행하는 모델로 전환
4. **인시던트 대응 준비**: 더 빠르고 자동화된 공격 패턴을 고려한 플레이북 업데이트

오펜시브와 디펜시브 AI 역량의 격차는 고정된 것이 아닙니다. 방어적 도구가 성숙해질수록 좁혀집니다. 지금 보안 자동화를 시작하지 않는 팀은 나중에 이 격차를 좁히기 더 어렵다는 것을 발견하게 될 것입니다.
