Anthropic이 Claude Opus 4.7을 출시했습니다. 14개 벤치마크 중 12개에서 최고 점수를 기록했으며, 가격은 Opus 4.6과 동일한 $5/$25 per million tokens를 유지합니다. 성능 수치도 인상적이지만, 더 주목할 부분은 Anthropic이 이 모델을 어떻게 활용할 것을 권장하는가에 있습니다.

## '페어 프로그래머'에서 '위임 엔지니어'로

Anthropic의 Opus 4.7 가이드라인은 이 모델을 "페어 프로그래머"가 아닌 "위임 엔지니어(delegated engineer)"로 다룰 것을 명시합니다. 이 구분은 실무에서 중요한 의미를 가집니다.

페어 프로그래머는 매 단계마다 지시를 기다립니다. 위임 엔지니어는 목표를 받아 스스로 판단하며 결과를 완성합니다. Anthropic은 에이전틱 코딩 워크플로우의 기본값으로 새로운 `xhigh` effort level을 권장합니다. 이 설정은 중간 확인 없이 여러 단계에 걸친 작업을 끝까지 처리하도록 설계되어 있습니다.

단계별 마이크로 지시로 Claude를 사용하고 있다면, 모델의 실제 능력을 충분히 활용하지 못하고 있는 것입니다.

## Claude Code 업데이트 사항

모델 릴리즈와 함께 Claude Code에도 실질적인 개선이 이루어졌습니다.

**Worktree Isolation**

병렬 코딩 작업이 독립된 Git Worktree에서 실행됩니다. 여러 기능을 동시에 작업하더라도 브랜치 간 충돌이 발생하지 않습니다.

```bash
# Claude Code가 내부적으로 관리하는 개념적 예시
git worktree add .worktrees/feature-auth -b feature/auth
git worktree add .worktrees/feature-api -b feature/api
# 각 Worktree에서 독립적으로 태스크 실행
```

**세션 간 메모리 관리 개선**

장기 프로젝트에서 컨텍스트 지속성이 향상되었습니다. 아키텍처 결정, 코딩 규약, 알려진 제약 조건처럼 세션 경계를 넘어 유지되어야 하는 정보는 영구 메모리에 저장되고, 임시 작업 컨텍스트는 적절히 정리됩니다. 대규모 코드베이스를 다루는 장기 프로젝트에서 출력 품질이 안정적으로 유지됩니다.

## Agentic 워크플로우를 위한 태스크 설계

위임 엔지니어링의 효과는 태스크를 어떻게 정의하느냐에 크게 달려 있습니다.

### 절차가 아닌 목표를 설정하세요

Claude에게 개별 변경 사항을 일일이 지시하는 대신, 원하는 결과를 설명하고 경로는 모델이 결정하게 하세요.

```
# 효과적인 방식 (목표 중심)
태스크: 주문 목록 페이지에서 N+1 쿼리 문제가 발생하고 있습니다.
100개 레코드 표시 시 200개 이상의 DB 쿼리가 실행됩니다.
근본 원인을 조사하고 수정하며, 영향받는 테스트도 업데이트하세요.
완료 기준: 페이지 로드 500ms 이하, 모든 테스트 통과.

# 덜 효과적인 방식 (과도한 세분화)
1. OrderRepository를 열어서
2. findAll 메서드를 찾아서
3. include로 eager loading 추가하고...
```

### 컨텍스트를 미리 제공하세요

위임 엔지니어가 올바른 자율 판단을 내리려면 초반에 충분한 컨텍스트가 필요합니다. 아키텍처 제약 사항, 코딩 규약, 관련 문서 링크를 작업 시작 시 미리 제공하면 중간 확인 횟수를 크게 줄일 수 있습니다.

### 완료 기준을 명확히 정의하세요

명확한 인수 기준이 있으면 모델이 스스로 검증할 수 있습니다.

```
완료 조건:
- npm test 전체 통과
- npm run typecheck 오류 없음
- 수정된 Endpoint의 응답 시간 기록 (before/after)
```

## 현재 한계에 대한 솔직한 평가

에이전틱 코딩이 인간의 판단을 완전히 대체하지는 않습니다. 여전히 신중한 처리가 필요한 영역이 있습니다.

**도메인 지식 공백**: 코드에 담겨 있지 않은 비즈니스 로직과 조직 고유의 제약 사항은 핵심 판단 시점에 인간의 개입이 필요합니다.

**보안 리뷰**: AI가 생성한 코드도 보안 관점에서 반드시 검토해야 합니다. 테스트로 잡히지 않는 미묘한 취약점이 도입될 수 있습니다.

**광범위한 의존성 변경**: 영향 범위가 넓은 메이저 버전 업그레이드는 단순 기능 작업보다 더 많은 감독이 필요합니다.

현재 사이버보안 분야 파트너들에게 제한적으로 공개된 Claude Mythos는 32단계 사이버 레인지를 10회 중 3회 해결했습니다. 일반적인 코딩 워크플로우는 이보다 훨씬 다루기 쉬운 작업이지만, 핵심 원칙은 동일합니다. 결과의 질은 인간 측의 협업 구조 설계에 달려 있습니다.

## 정리

`xhigh` effort level과 업데이트된 Claude Code를 활용한 Claude Opus 4.7은 AI를 도구에서 팀 구성원으로 전환하는 의미 있는 단계를 나타냅니다. 최대한 활용하려면:

- 절차가 아닌 목표를 할당하세요
- 충분한 컨텍스트를 초반에 제공하세요
- 검증 가능한 완료 기준을 정의하세요
- 보안에 민감한 변경 사항은 직접 검토하세요

기술은 이미 에이전틱 워크플로우에 준비되어 있습니다. 제한 요소는 대부분 인간 측의 태스크 설계에 있습니다.
