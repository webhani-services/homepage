## Claude Opus 4.7 개요

Claude Opus 4.7은 2026년 4월 16일에 출시되었습니다. 이번 릴리즈의 핵심은 SWE-bench Pro 점수입니다. 이전 버전 대비 10.9포인트 향상되어, 1,865개의 테스트 세트에서 약 200개 이상의 소프트웨어 엔지니어링 작업을 자율적으로 해결할 수 있게 되었습니다.

SWE-bench Pro는 실제 GitHub Issue를 AI가 자율적으로 해결하는 능력을 측정하는 벤치마크입니다. 단순한 코드 자동완성이 아니라, 문제 이해 → 관련 코드 조사 → 변경 구현 → 테스트 통과까지의 전체 엔지니어링 사이클을 자율적으로 수행할 수 있는지 평가합니다.

## Agentic Workflow로의 패러다임 전환

같은 시기에 개최된 Code with Claude 2026 컨퍼런스에서는 더 중요한 메시지가 전달되었습니다. "LLM을 사용하는 것"에서 "LLM이 작동하는 시스템을 설계하는 것"으로의 전환이 주요 테마였습니다.

에이전트 워크플로우의 특성은 다음과 같습니다:

- **Goal Orientation**: 최종 목표에서 역산하여 하위 작업을 자율적으로 계획·실행
- **Memory Retention**: 실행 컨텍스트를 단계를 넘어 유지·참조
- **External System Interaction**: API 호출, 코드 실행, 파일 조작 등 외부 액션 수행
- **Self-Correction**: 실행 결과를 평가하고 목표에 맞게 계획 수정

기존의 "질문 → 답변" 모델에서 "목표 → 계획 → 실행 → 검증 → 완료"의 루프 구조로 변화하고 있습니다.

## 실용적인 에이전트 설계 패턴

### Plan → Execute → Verify 루프

```python
import anthropic
from typing import Any

client = anthropic.Anthropic()

TOOLS = [
    {
        "name": "run_command",
        "description": "Run a shell command and return output",
        "input_schema": {
            "type": "object",
            "properties": {
                "command": {"type": "string"}
            },
            "required": ["command"]
        }
    },
    {
        "name": "read_file",
        "description": "Read file contents",
        "input_schema": {
            "type": "object",
            "properties": {
                "path": {"type": "string"}
            },
            "required": ["path"]
        }
    }
]

def engineering_agent(task: str, max_steps: int = 20) -> str:
    messages: list[dict] = [{"role": "user", "content": task}]

    for _ in range(max_steps):
        response = client.messages.create(
            model="claude-opus-4-7-20260416",
            max_tokens=8192,
            tools=TOOLS,
            messages=messages
        )

        messages.append({"role": "assistant", "content": response.content})

        # 에이전트가 완료를 판단한 경우
        if response.stop_reason == "end_turn":
            return next(
                (b.text for b in response.content if hasattr(b, "text")),
                "완료"
            )

        # Tool 결과를 수집하고 다시 전달
        tool_results = []
        for block in response.content:
            if block.type == "tool_use":
                result = handle_tool(block.name, block.input)
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": result
                })

        if tool_results:
            messages.append({"role": "user", "content": tool_results})

    return "최대 단계 도달"


def handle_tool(name: str, params: dict[str, Any]) -> str:
    import subprocess

    if name == "run_command":
        result = subprocess.run(
            params["command"], shell=True,
            capture_output=True, text=True, timeout=30
        )
        return (result.stdout + result.stderr)[-2000:]

    if name == "read_file":
        with open(params["path"]) as f:
            return f.read()

    return f"알 수 없는 Tool: {name}"
```

`stop_reason == "end_turn"`일 때 에이전트가 자율적으로 "완료"를 판단합니다. Tool을 계속 호출할지, 텍스트로 결론을 내릴지는 모델이 결정합니다. Tool 출력을 2,000자로 잘라내는 것은 컨텍스트 윈도우 관리를 위한 중요한 패턴입니다.

### 상태 영속성

멀티스텝 작업에서는 중간 상태 관리가 중요합니다.

```python
import json
from pathlib import Path

def save_checkpoint(session_id: str, messages: list, completed: list[str]) -> None:
    path = Path(f".sessions/{session_id}.json")
    path.parent.mkdir(exist_ok=True)
    path.write_text(json.dumps({
        "messages": messages,
        "completed": completed
    }, ensure_ascii=False, indent=2))

def load_checkpoint(session_id: str) -> dict | None:
    path = Path(f".sessions/{session_id}.json")
    if not path.exists():
        return None
    return json.loads(path.read_text())
```

세션이 중단되어도 `messages` 히스토리를 복원하여 에이전트 루프를 재시작할 수 있습니다.

## 주요 AI 코딩 도구와의 관계

현재 Claude는 Cursor, Windsurf, Claude Code 세 가지 주요 AI 코딩 도구의 백엔드로 작동하고 있습니다. Opus 4.7의 향상된 성능은 이러한 도구들을 통해 직접적으로 체감할 수 있습니다:

- **Claude Code**: CLI 완전 통합형. Git 작업, 파일 변경, 테스트 실행까지 통합
- **Cursor**: 에디터 통합형. 전체 코드베이스를 컨텍스트로 포함한 보완
- **Windsurf**: 흐름 기반. 연속적인 구현 작업을 흐르듯 처리

대규모 리팩토링이나 여러 파일에 걸친 버그 수정이 이전보다 높은 정확도로 완료됩니다. 예전에는 "어떤 파일을 수정해야 하나요?", "어떤 테스트를 실행해야 하나요?"와 같은 확인 프롬프트가 여러 번 필요했던 작업들이 중단 없이 완료되는 경우가 늘었습니다.

## 프로덕션 배포 시 3가지 원칙

**1. 반복 횟수 제한**: 제어 없는 에이전트 루프는 비용과 실행 시간의 초과를 초래합니다. `max_steps`는 반드시 설정하고, 상한에 도달했을 때 알림을 내보내는 구조도 함께 구현하세요.

**2. 코드 실행 샌드박스화**: `run_command` 등의 Tool을 제공할 경우, Docker 컨테이너나 gVisor 등의 샌드박스 환경에서 실행해야 합니다. 네트워크 액세스를 제한하고, 프로덕션 데이터에는 읽기 전용 마운트를 기본으로 설정하세요.

**3. Human-in-the-loop 설계**: 프로덕션 데이터 쓰기, 외부 API 전송 등 되돌릴 수 없는 작업에는 확인 단계를 두세요. 에이전트가 자율적으로 여러 단계를 실행할 때 잘못된 행동이 연쇄될 위험이 있습니다.

## 정리

Claude Opus 4.7의 SWE-bench 개선은 엔지니어링 작업의 자율 실행이 실용적인 수준에 가까워지고 있음을 보여줍니다.

하지만 능력 향상이 "무엇이든 맡길 수 있다"를 의미하지는 않습니다. 샌드박스, 비용 관리, Human-in-the-loop가 프로덕션 운용의 기반입니다. 에이전트 시스템을 자사 서비스에 통합할 때는 이 세 가지를 설계의 출발점으로 삼을 것을 권장합니다.
