2026년 4월 기준으로 Model Context Protocol(MCP)은 LLM과 외부 도구를 연결하는 사실상의 표준 인터페이스로 자리잡았습니다. Database 접근, Cloud 인프라 관리, Browser 자동화, 개발 도구 통합 등 모든 영역에서 MCP Server 구현체가 등장하고 있습니다.

MCP가 빠르게 확산된 이유는 단순합니다. LLM이 외부 도구를 호출하는 방식을 표준화하여, 모델이 바뀌어도 도구가 바뀌어도 연결 레이어는 그대로 유지됩니다.

## MCP 동작 방식

MCP는 클라이언트-서버 모델입니다. LLM이 MCP Client로 동작하고, 각 도구는 MCP Server로 제공됩니다. 통신은 JSON-RPC over stdio 또는 HTTP로 이루어집니다.

```
LLM (MCP Client)
    ↓ JSON-RPC over stdio / HTTP
MCP Server
    ↓
외부 도구 (DB, API, 브라우저 등)
```

MCP Server가 노출하는 리소스 유형은 세 가지입니다.

- **Tools**: LLM이 호출할 수 있는 함수 (쿼리 실행, 파일 조작 등)
- **Resources**: LLM이 참조할 수 있는 데이터 (스키마, 문서 등)
- **Prompts**: 재사용 가능한 프롬프트 템플릿

## TypeScript로 구현하는 최소 MCP Server

읽기 전용 PostgreSQL MCP Server 예시입니다.

```typescript
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { Pool } from "pg";

const pool = new Pool({ connectionString: process.env.DATABASE_URL });

const server = new Server(
  { name: "postgres-readonly", version: "1.0.0" },
  { capabilities: { tools: {} } }
);

server.setRequestHandler("tools/list", async () => ({
  tools: [
    {
      name: "query",
      description: "Read-only SQL 쿼리를 실행합니다",
      inputSchema: {
        type: "object",
        properties: {
          sql: { type: "string" },
        },
        required: ["sql"],
      },
    },
  ],
}));

server.setRequestHandler("tools/call", async ({ params }) => {
  if (params.name !== "query") throw new Error("Unknown tool");

  const sql = params.arguments.sql as string;
  if (!/^\s*SELECT\b/i.test(sql)) {
    return { content: [{ type: "text", text: "SELECT 쿼리만 허용됩니다" }] };
  }

  const result = await pool.query(sql);
  return {
    content: [{ type: "text", text: JSON.stringify(result.rows, null, 2) }],
  };
});

const transport = new StdioServerTransport();
await server.connect(transport);
```

## 2026년 4월 MCP 생태계

서버 생태계가 빠르게 성장하고 있습니다. 카테고리별 현황입니다.

**Database**: PostgreSQL, MySQL, SQLite, MongoDB, Redis, Supabase, PlanetScale

**Cloud Infrastructure**: AWS(EC2, S3, CloudWatch, EKS), Kubernetes Pod 관리·로그 조회, Terraform State 조회

**Browser & Web**: Playwright, Puppeteer, Web 스크래핑, Form 자동화

**Developer Tools**: GitHub/GitLab(Issue, PR, Actions), Jira, Linear, Slack, Discord

DevOps 분야에서만 18개 이상의 활성 MCP Server가 공개되어 있으며, CI/CD 파이프라인에 LLM을 통합하는 것이 연구 과제가 아닌 실용적인 엔지니어링 선택지가 되었습니다.

## Claude Code에서 MCP 활용하기

Claude Code는 설정 파일을 통해 MCP Server를 기본으로 지원합니다.

```json
// .claude/settings.json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_CONNECTION_STRING": "postgresql://localhost/mydb"
      }
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "ghp_***"
      }
    }
  }
}
```

이 설정으로 Claude Code는 쿼리를 생성하면서 Database 스키마를 직접 참조하거나, 코드를 수정하면서 GitHub Issue를 확인할 수 있습니다. 컨텍스트를 수동으로 복사-붙여넣기할 필요가 없습니다.

## 보안 고려사항

MCP Server는 LLM이 영향을 미칠 수 있는 범위를 확장하므로 접근 제어가 중요합니다.

- **최소 권한 원칙**: 쿼리 도구에는 읽기 전용 DB 사용자 사용
- **시크릿 관리**: 자격증명은 환경 변수에 저장, 소스 코드에 포함하지 않기
- **입력 검증**: LLM이 생성한 SQL을 파싱·검증 없이 DB에 직접 전달하지 않기
- **감사 로그**: 원래 프롬프트와 함께 모든 MCP 도구 호출 기록

## 정리

MCP의 역할은 LLM 생태계에서 HTTP가 웹 API에 했던 것과 유사합니다. 커뮤니티가 공통 인터페이스를 기반으로 구축할 수 있게 해줍니다. 도구 개발자는 서버를 한 번 작성하면 되고, Agent 개발자는 어떤 모델을 사용하든 모든 서버를 활용할 수 있습니다. 이 관심사의 분리가 생태계 성장을 가속화하는 이유입니다. AI 워크플로우를 구축하고 있다면, MCP를 통한 통합이 커스텀 도구 구현보다 장기적으로 유지보수에 유리합니다.
