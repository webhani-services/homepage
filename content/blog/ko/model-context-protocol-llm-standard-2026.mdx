## MCP란 무엇인가

Model Context Protocol(MCP)은 LLM 애플리케이션과 외부 도구 및 데이터 소스를 연결하는 오픈 프로토콜입니다. Anthropic이 사양을 공개한 이후, 2026년 들어 채택이 급속히 확산되었습니다. 현재는 Database 접근, 클라우드 인프라 관리, 브라우저 자동화 등 다양한 MCP Server가 등장했습니다.

핵심 개념은 간단합니다. MCP는 LLM이 외부 도구를 호출하는 표준 방식을 제공합니다. LLM 애플리케이션과 도구 모두 MCP를 한 번 구현하면, 별도의 커스텀 통합 코드 없이 상호 운용이 가능해집니다.

## MCP가 표준으로 자리잡는 이유

### Provider 독립성

MCP는 특정 LLM 벤더에 종속되지 않습니다. Claude, GPT-5.5, Gemini 어느 모델을 사용하더라도 동일한 MCP Server에 연결할 수 있습니다. 모델을 전환하거나 여러 모델을 병렬로 운용하는 팀에게 특히 중요한 장점입니다.

### 빠른 Ecosystem 성장

2026년 4월 기준으로, 주요 Database(PostgreSQL, MySQL, MongoDB), 클라우드 플랫폼(AWS, GCP, Azure), 개발 도구(GitHub, Linear, Jira)에 대한 공식 또는 잘 관리된 MCP Server가 존재합니다. 새로운 데이터 소스에 LLM을 연결하는 데 걸리는 시간이 며칠에서 몇 시간으로 줄었습니다.

### 명확한 권한 Scope

MCP는 도구가 접근할 수 있는 권한 범위를 명시적으로 선언하도록 요구합니다. LLM이 어떤 리소스에 접근할 수 있는지 명확하게 정의되기 때문에, 임시방편적인 Function Calling 방식보다 보안 감사가 용이합니다.

## 구현 예시

Node.js로 작성한 MCP Server의 기본 구조입니다.

```typescript
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({
  name: "internal-api-server",
  version: "1.0.0",
});

server.tool(
  "get_product",
  "상품 ID로 상품 정보를 조회합니다",
  { productId: z.string() },
  async ({ productId }) => {
    const product = await db.products.findById(productId);
    return {
      content: [{ type: "text", text: JSON.stringify(product) }],
    };
  }
);

const transport = new StdioServerTransport();
await server.connect(transport);
```

Client 측에서는 LLM 애플리케이션의 설정에 MCP Server를 등록하기만 하면 됩니다.

```json
{
  "mcpServers": {
    "internal-api": {
      "command": "node",
      "args": ["./mcp-servers/api-server.js"],
      "env": {
        "API_BASE_URL": "${API_BASE_URL}"
      }
    }
  }
}
```

커스텀 통합 코드를 별도로 작성할 필요가 없습니다.

## MCP가 가장 효과적인 상황

**여러 LLM Agent가 동일한 도구에 접근해야 할 때**: MCP Server를 한 번 구현하면 모든 Agent에서 호출 가능합니다. Agent별 개별 통합 코드가 필요 없습니다.

**모델 전환을 고려하는 경우**: MCP는 프로토콜 레이어에서 추상화되어 있어, Claude에서 GPT-5.5로 전환해도 도구 통합 코드를 재작성할 필요가 없습니다.

**감사 가능성이 중요한 경우**: 금융, 의료 등 규제가 엄격한 분야에서는 LLM이 어떤 데이터에 접근했는지 추적이 필요합니다. MCP의 명시적 권한 모델은 이러한 요구사항에 잘 부합합니다.

## 과도한 설계가 될 수 있는 경우

도구 호출이 한두 가지에 불과한 단순한 챗봇이라면, MCP 도입 비용이 이점을 상회할 수 있습니다. 먼저 직접적인 Function Calling으로 시작하고, 여러 Agent나 모델이 관여하거나 통합 범위가 확장될 때 MCP를 도입하는 것이 현실적입니다.

## 정리

MCP는 REST API가 2010년대 초 웹 서비스 통합을 표준화했던 것처럼, LLM 도구 통합을 표준화하고 있습니다. Ecosystem이 이미 성숙해진 지금, 내부 시스템에 접근해야 하는 새로운 LLM 애플리케이션을 설계할 때는 MCP를 기본 아키텍처로 검토하는 것을 권장합니다. 재사용성과 유지보수성이 크게 향상됩니다.
