Next.js 15.5에는 실무에 직접 영향을 미치는 세 가지 변경 사항이 포함되어 있습니다. Node.js Middleware 안정화, Turbopack Production Build Beta 제공, TypeScript DX 개선입니다. 각 변경의 실질적인 의미를 정리합니다.

## Node.js Middleware 안정화

기존 Next.js Middleware는 Edge Runtime에서만 동작했습니다. Edge Runtime은 빠르고 가볍지만 Node.js API를 사용할 수 없다는 제약이 있었습니다. `fs`, `crypto`, 네이티브 모듈, 무거운 인증 라이브러리 등을 Middleware에서 직접 사용할 수 없었습니다.

15.5부터 Middleware가 Node.js Runtime에서 안정적으로 동작합니다. Node.js의 전체 API를 Middleware에서 사용할 수 있습니다.

```typescript
// middleware.ts — Node.js API를 직접 사용 가능
import { NextRequest, NextResponse } from 'next/server'
import { createHash } from 'crypto' // 이전에는 Middleware에서 사용 불가

export async function middleware(request: NextRequest) {
  const authHeader = request.headers.get('authorization')

  if (!authHeader?.startsWith('Bearer ')) {
    return NextResponse.json({ error: 'Token이 없습니다' }, { status: 401 })
  }

  const token = authHeader.slice(7)

  // Middleware에서 직접 crypto 모듈 사용 가능
  const tokenHash = createHash('sha256').update(token).digest('hex')
  const isValid = await validateTokenHash(tokenHash)

  if (!isValid) {
    return NextResponse.redirect(new URL('/login', request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ['/api/:path*', '/dashboard/:path*']
}
```

가장 큰 영향을 받는 영역은 인증입니다. JWT 검증, 세션 확인, 암호화 처리 등을 위해 API Route를 거쳐야 했던 로직을 Middleware에 직접 구현할 수 있게 되었습니다.

## Turbopack Production Build

`next build --turbopack` 명령으로 Production Build에 Turbopack을 사용할 수 있습니다. Vercel은 이미 vercel.com, v0.app, nextjs.org에 적용하고 있습니다.

```bash
# Turbopack으로 Production Build
next build --turbopack
```

```json
// package.json에 추가
{
  "scripts": {
    "build": "next build --turbopack"
  }
}
```

Build 속도 개선 폭은 프로젝트 규모와 구성에 따라 다릅니다. 대규모 코드베이스일수록 효과가 큽니다.

**주의**: Beta 단계입니다. 일부 Webpack Plugin이나 커스텀 설정과 호환성 문제가 있을 수 있습니다. Production 적용 전에 CI 환경에서 기존 설정과의 호환성을 반드시 확인하세요.

## TypeScript DX 개선

### Typed Routes 안정화

Link `href`가 컴파일 시점에 검증됩니다. 존재하지 않는 Route를 지정하면 Type Error가 발생합니다.

```typescript
// /products/[id] Route가 있으면 통과
<Link href="/products/123">상품 보기</Link>

// Route가 없으면 Compile Error
<Link href="/productz/123">오타</Link>
```

Config에서 활성화합니다.

```typescript
// next.config.ts
const nextConfig = {
  experimental: {
    typedRoutes: true // 15.5에서 Stable
  }
}
```

### Helper Types: PageProps, LayoutProps

```typescript
// app/blog/[slug]/page.tsx
import type { PageProps } from 'next/types'

export default function BlogPost({ params, searchParams }: PageProps) {
  // params, searchParams 자동 타입 추론 — 수동 타입 정의 불필요
  return <article>{params.slug}</article>
}

export async function generateMetadata({ params }: PageProps) {
  return { title: `포스트: ${params.slug}` }
}
```

### next typegen 명령어

전체 Build 없이 Type Artifact만 생성할 수 있는 새 명령어입니다.

```bash
# .next/types 생성 — 전체 Build보다 빠름
npx next typegen
```

CI에서 Build 없이 Type Check만 수행할 때 유용합니다.

## 기존 프로젝트 마이그레이션 고려사항

`runtime = 'edge'`를 명시적으로 지정한 Middleware가 있다면 제거하거나 업데이트해야 합니다.

```typescript
// Node.js Runtime 사용 시 이 줄을 제거
export const runtime = 'edge'
```

Typed Routes를 기존 프로젝트에 활성화하면 깨진 Link가 한 번에 노출됩니다. 먼저 `npx next typegen`으로 범위를 확인하고 단계적으로 수정하세요.

## webhani의 관점

Node.js Middleware 안정화는 15.5에서 가장 실용적인 변경 사항입니다. 인증 로직을 위해 Middleware에서 API Route로 우회하던 패턴을 제거할 수 있습니다. 코드 구조가 단순해지고 Latency도 줄어듭니다.

Turbopack Production Build는 Staging 환경에서 먼저 테스트하고, Beta 졸업 후 Production 전환 계획을 세우는 접근을 권장합니다.
