## 먼저 보안 패치 이야기부터

2026년 3월 18일, Next.js 16.2가 릴리즈되었습니다. 새 기능을 살펴보기 전에 보안 상황을 먼저 다뤄야 합니다.

Next.js 15.x 및 16.x에 영향을 미치는 CVSS 10.0 취약점이 확인되었습니다. React Server Components 프로토콜과 관련된 문제로, 원격 코드 실행(RCE)이 가능할 수 있습니다. 두 버전 중 하나를 프로덕션에서 사용하고 있다면 16.2로의 업그레이드는 선택이 아닙니다.

```bash
# 현재 버전 확인
npx next --version

# 업그레이드
npm install next@latest react@latest react-dom@latest
```

업그레이드는 스테이징 환경에서 먼저 진행하고 TypeScript 타입 오류가 없는지 확인하세요. 16.2는 일부 영역에서 타입 검사가 더 엄격해져 기존에 통과하던 오류가 새로 발견될 수 있습니다.

## 실험적 기능: AI Agent DevTools

이번 릴리즈에서 가장 주목받은 기능입니다. AI Agent DevTools는 Claude Code 같은 AI 코딩 에이전트가 개발 중에 React DevTools와 Next.js 진단 정보에 터미널 수준으로 접근할 수 있도록 합니다.

새로 추가된 Browser Log Forwarding 기능과 함께 사용하면, 클라이언트 사이드 콘솔 출력이 에이전트의 터미널 세션으로 전달됩니다. AI 에이전트가 프런트엔드 런타임 오류를 스크린샷이나 수동 복사 없이 직접 확인할 수 있게 됩니다.

```typescript
// next.config.ts
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    agentDevTools: true,        // AI 에이전트에게 DevTools 진단 정보 접근 허용
    browserLogForwarding: true, // 브라우저 콘솔을 터미널로 전달
  },
}

export default nextConfig
```

프로덕션에서는 비활성화하세요. 이 기능의 가치는 개발 루프에 있습니다. 실행 중인 애플리케이션과 그 위에서 작업하는 AI 에이전트 사이의 Context 전달을 빠르게 만드는 것이 목적입니다.

## Server Fast Refresh

Turbopack이 Server Component의 세밀한 Hot Reloading을 지원하게 되었습니다. 기존에는 Server Component를 수정하면 페이지 전체가 다시 렌더링되었지만, Server Fast Refresh는 변경된 컴포넌트 트리만 업데이트합니다.

```
이전: Server Component 수정 → 페이지 전체 재렌더링
이후: Server Component 수정 → 변경된 서브트리만 업데이트
```

Server Component 트리가 깊은 대규모 애플리케이션에서 파일 저장 후 결과 확인까지의 시간이 의미 있게 단축됩니다. 별도 설정 변경 없이 `next.config.ts`에서 Turbopack을 활성화한 것만으로 동작합니다.

## Web Worker Origin

WASM 라이브러리의 Worker 지원이 개선되었습니다. Web Worker 내에서 WASM 모듈을 로드할 때 발생하던 오리진 제약이 완화되어, 이미지 처리·암호화·오디오 파이프라인 등 브라우저 샌드박스 환경을 활용하는 라이브러리와의 통합이 쉬워졌습니다.

Worker에서 WASM을 사용하지 않는다면 이 변경사항은 해당되지 않습니다.

## 개발 서버 Lock File

작지만 실용적인 추가 기능입니다. `next dev` 시작 시 `.next/dev.lock`에 Lock File을 생성합니다. 여러 개발 서버 프로세스가 같은 포트에서 충돌하는 경우를 방지합니다.

```bash
# 개발 서버가 비정상 종료되어 Lock File이 남아있는 경우
rm -f .next/dev.lock
```

## create-next-app 템플릿 업데이트

`create-next-app`의 기본 프로젝트 템플릿이 AI 에이전트 친화적인 구조로 업데이트되었습니다. 새 프로젝트는 AI 에이전트가 탐색하고 추론하기 쉬운 파일 구조로 스캐폴딩됩니다.

```bash
npx create-next-app@latest my-app
```

## TypeScript 타입 검사 강화

16.2는 일부 영역에서 타입 검사를 강화했습니다. 기존 프로젝트를 업그레이드하면 이전에는 통과하던 오류가 새로 나타날 수 있습니다.

주의가 필요한 부분:

- Async Request API(`await cookies()`, `await headers()`, `await params`)의 타입 정의가 더 엄격해짐
- Route Handler 반환 타입 검사 강화
- Server Action 인수 타입 검증 강화

특히 Async Request API 주변에 `any` 타입 처리가 누적된 프로젝트라면 업그레이드 시 타입 수정 작업을 예상하세요.

## 업그레이드 체크리스트

1. **보안 패치를 먼저 적용** — 다른 변경 전 16.2로 업그레이드
2. **`experimental` 플래그 검토** — 일부 플래그 이름이 변경되었으므로 마이그레이션 노트 확인
3. **새로 발생한 TypeScript 오류 수정** — 실제 문제를 잡아내는 변경사항이므로 무시하지 말 것
4. **Turbopack 설정 확인** — `experimental.turbopack`이 아닌 최상위 `turbopack` 설정이 적용되어 있는지 확인
5. **Server Component Hot Reload 검증** — 개발 환경에서 Server Fast Refresh 정상 동작 확인

## 정리

보안 패치만으로도 16.2는 Next.js 15.x 또는 16.x를 프로덕션에서 운영 중인 모든 팀에게 필수 업그레이드입니다. AI Agent DevTools와 Server Fast Refresh는 AI 코딩 에이전트를 이미 사용 중인 팀의 개발 워크플로우를 실질적으로 개선합니다. 스테이징에서 먼저 적용하고, 타입 오류를 수정한 뒤 프로덕션에 반영하세요.
