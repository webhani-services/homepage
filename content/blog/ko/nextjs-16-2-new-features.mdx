# Next.js 16.2: AI-Ready create-next-app, Turbopack Server Fast Refresh, SRI 지원

Next.js 16.2가 공개되었습니다. AI 개발 워크플로우 지원 강화, Turbopack의 서버사이드 Hot Reloading 개선, JavaScript 자산의 Subresource Integrity 지원이 주요 변경사항입니다.

## AI-Ready create-next-app

`create-next-app`이 AI Code Generator와 Agent에 최적화되었습니다.

```bash
npx create-next-app@latest my-app
```

주요 추가 기능:

- **Browser Log Forwarding**: 브라우저 콘솔 출력이 터미널로 스트리밍되어 AI Agent가 별도 도구 없이 브라우저 상태를 확인할 수 있습니다
- **Dev Server Lock File Handling**: 여러 Agent가 동일 프로젝트를 동시에 작업할 때 발생하는 Race Condition을 방지합니다
- **Experimental Agent DevTools**: AI Agent가 앱 상태를 직접 검사하고 상호작용할 수 있는 초기 단계의 도구입니다

개발 도구가 인간 단독 작업이 아닌 인간-AI 협업을 위해 설계되는 방향을 보여주는 변화입니다.

## Turbopack Server Fast Refresh

Turbopack에 세밀한 서버사이드 Hot Reloading이 추가되었습니다. Server Component 변경 시 전체 페이지를 재렌더링하는 대신 변경된 Component만 업데이트합니다.

이전에는 Server Component를 수정할 때마다 페이지 전체가 새로고침되면서 폼 입력값이나 스크롤 위치 같은 클라이언트 상태가 초기화되었습니다. Server Fast Refresh는 그 상태를 유지하면서 변경된 Component만 업데이트합니다.

데이터 조회가 많은 복잡한 Server Component 트리를 가진 페이지에서 수정-확인 사이클이 빨라집니다.

## Web Worker Origin과 WASM 지원

Turbopack의 Web Worker Origin으로 Worker에서 WASM 라이브러리 지원이 확대되었습니다.

```typescript
// Worker에서 WASM 실행
const worker = new Worker(new URL('./heavy-computation.worker.ts', import.meta.url));
```

이미지 처리, 암호화 연산, 데이터 파싱 등 CPU 집약적인 작업을 메인 스레드에서 분리하는 데 유용합니다.

## JavaScript의 Subresource Integrity

JavaScript 자산에 SRI Hash가 자동으로 추가됩니다.

```html
<script
  src="/_next/static/chunks/main.js"
  integrity="sha256-abc123..."
  crossorigin="anonymous"
></script>
```

CDN이나 서드파티 Script 배포가 침해된 경우, 브라우저가 예상 Hash와 일치하지 않는 파일 실행을 차단합니다. 엔터프라이즈나 공공기관 대상 애플리케이션의 보안 요구사항을 충족하는 데 도움이 됩니다.

## 보안 수정

16.2에는 여러 고·중·저 심각도 취약점 수정과 Prerender Recovery, Route Params, Cache Key 처리에 대한 핵심 버그 수정이 포함되어 있습니다. 기존 프로젝트는 조속히 업그레이드하시기 바랍니다.

```bash
npm install next@latest
```

## 정리

16.2의 Agent DevTools와 Browser Log Forwarding은 Next.js가 AI Agent를 단순한 도구가 아닌 공동 작업자로 설계하는 방향을 명확히 보여줍니다. Server Fast Refresh와 SRI는 오래된 개발자 요구사항을 해결합니다. 보안 수정만으로도 업그레이드할 이유가 충분합니다.
