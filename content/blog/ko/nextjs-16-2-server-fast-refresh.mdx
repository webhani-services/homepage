Next.js 16.2가 출시되었습니다. 이번 릴리스는 화려한 신기능보다 개발자 경험 개선에 집중했으며, 그 중 핵심은 **Server Fast Refresh**입니다. Server Components에 세밀한 핫 리로드를 제공하는 기능입니다.

## Server Fast Refresh란

이전까지 Server Component를 수정하면 페이지 전체가 리로드되는 경우가 많았습니다. 스크롤 위치, 폼 입력값, 포커스된 요소들이 모두 초기화되는 불편함이 있었습니다. 작은 변경을 자주 가하는 리팩토링 세션에서 특히 거슬리는 부분이었습니다.

Server Fast Refresh는 이 문제를 해결합니다. Server Component를 수정할 때 영향받는 서브트리만 재렌더링되고, 나머지 페이지 상태는 유지됩니다.

```typescript
// app/dashboard/page.tsx — Server Component
// 이 파일을 수정해도 전체 페이지 상태가 초기화되지 않습니다
export default async function DashboardPage() {
  const metrics = await fetchMetrics();
  return (
    <section>
      <MetricsGrid data={metrics} />
      <ActivityFeed /> {/* 마운트 유지, 스크롤 상태 보존 */}
    </section>
  );
}
```

현재 App Router 모범 사례에 따라 대부분의 로직을 Server Component에 두는 팀이라면, 리팩토링 작업의 체감 속도가 개선됩니다.

## Turbopack 주요 업데이트

16.2에는 200개 이상의 Turbopack 변경사항이 포함되어 있습니다. 주목할 부분들:

**Web Worker Origin** 문제가 해결되었습니다. WASM 라이브러리 사용 시 Worker가 잘못된 오리진으로 실행되던 호환성 문제가 수정되었습니다. WebAssembly 모듈을 사용하면서 원인 불명의 버그를 겪었다면, 이번 업데이트로 해결될 수 있습니다.

**Subresource Integrity (SRI) 지원**이 Turbopack 수준에서 추가되었습니다. CDN을 통해 제공되는 스크립트의 무결성 검증이 가능해져, 엄격한 Content Security Policy가 적용된 프로젝트에 유용합니다.

**Dynamic import 트리 쉐이킹** 개선:

```typescript
// import()로 로드되는 모듈의 불필요한 코드가 더 적극적으로 제거됩니다
const { AnalyticsChart } = await import('./AnalyticsChart');
// AnalyticsChart의 모듈 그래프에서 사용되지 않는 의존성은
// 이제 더 안정적으로 번들에서 제외됩니다
```

별도 설정 변경 없이 대규모 애플리케이션의 프로덕션 번들 크기가 줄어드는 효과가 있습니다.

## AI-Ready 프로젝트 템플릿

`create-next-app`이 AI 통합 템플릿 선택을 지원합니다:

```bash
npx create-next-app@latest my-app
# AI 기능이 포함된 새 템플릿 옵션 추가
```

새 Next.js 프로젝트에서 AI SDK 통합이 이제 나중에 추가하는 것이 아니라 처음부터 고려하는 시작점이 되었음을 보여주는 변화입니다.

## Browser Log Forwarding

개발 중 브라우저 콘솔 로그가 터미널로 전달됩니다:

```
[browser] TypeError: Cannot read properties of undefined
[browser]   at ProductCard (components/ProductCard.tsx:23)
```

작은 변화이지만, 클라이언트 사이드 에러를 추적할 때 브라우저 DevTools와 터미널 사이를 왔다 갔다 하는 번거로움이 줄어듭니다. 서버 로그를 보고 있는 터미널에서 브라우저 에러도 바로 확인할 수 있습니다.

## Next.js 16 핵심 사항 재확인

16.2로 업그레이드한다면 16.0 마이그레이션에서 적용된 패턴도 계속 유효합니다:

**모든 Request API는 비동기:**

```typescript
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>;
}) {
  const { slug } = await params;
  const cookieStore = await cookies();
  // ...
}
```

**`proxy.ts`가 `middleware.ts`를 대체**합니다. Node.js 런타임에서만 실행되며 `app/`과 같은 레벨에 배치합니다:

```
src/
├── proxy.ts    ← 여기에 배치 (src/ 사용 시)
├── app/
│   └── ...
```

Edge 런타임 전용 API를 미들웨어에서 사용하고 있었다면 리팩토링이 필요합니다.

## 정리

Next.js 16.2는 16.0에서 도입된 패턴의 개발 경험을 개선하는 데 집중한 릴리스입니다. Server Fast Refresh는 특히 App Router 모범 사례를 따르는 팀, 즉 Server Component를 주로 사용하는 팀에게 실질적인 개선을 제공합니다.

새 API보다 빌드 도구와 개발 경험에 꾸준히 투자하는 것, 그것이 성숙한 프레임워크의 모습입니다. Next.js의 대규모 프로젝트 적용을 망설이게 했던 빌드 안정성 우려가 매 Turbopack 업데이트마다 체계적으로 해소되고 있습니다.
