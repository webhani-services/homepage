Next.js 16.2가 200개 이상의 변경 사항과 함께 출시됐습니다. 두 가지가 특히 주목할 만합니다. Request Smuggling 취약점 수정(CVE-2026-29057)과 Turbopack의 주요 개선입니다. 무엇을 알아야 하고 무엇을 해야 하는지 정리합니다.

## 보안 패치: CVE-2026-29057

이번 릴리스는 `http-proxy`의 Request Smuggling 취약점(CVE-2026-29057)을 패치합니다. Request Smuggling 공격은 Frontend Proxy와 Backend 서버가 HTTP 요청의 경계를 다르게 해석할 때 발생합니다. 이를 악용하면 Cache Poisoning, 인증 우회, 다른 사용자의 요청에 데이터를 혼입하는 등의 피해로 이어질 수 있습니다.

현재 Next.js 16.x를 Production에서 사용 중이라면 즉시 업그레이드할 것을 권장합니다.

```bash
# Next.js 및 React 업그레이드
npm install next@latest react@latest react-dom@latest

# 설치된 버전 확인
npx next --version
```

업그레이드 후에는 Server Actions Route와 Reverse Proxy를 통과하는 경로를 검증하세요. 특히 Vercel 외의 Self-hosted 환경에서는 Proxy 설정에 따라 동작이 다를 수 있습니다.

## 주목할 Turbopack 변경 사항

### Server Fast Refresh 개선

Server Component에 대한 HMR(Hot Module Replacement)이 빨라졌습니다. 서버 사이드 컴포넌트를 수정했을 때 개발 모드에서 피드백 루프가 눈에 띄게 짧아집니다. 기존 HMR 지연이 체감됐던 대규모 앱에서 특히 효과적입니다.

```typescript
// 이 Server Component를 수정하면 더 빠르게 Reload됩니다
export default async function ProductList() {
  const products = await fetchProducts();

  return (
    <ul>
      {products.map((product) => (
        <li key={product.id}>{product.name}</li>
      ))}
    </ul>
  );
}
```

### WASM 라이브러리용 Web Worker Origin 지원

Turbopack이 WASM 라이브러리를 사용하는 Web Worker의 Origin 설정을 처리할 수 있게 됐습니다. 이미지 처리, 암호화, 고성능 연산 등을 WASM으로 구현한 Worker를 Next.js 앱에 통합하기가 쉬워집니다.

### JavaScript 파일에 대한 Subresource Integrity(SRI) 지원

JavaScript 에셋에 SRI 지원이 추가됐습니다. 브라우저가 CDN이나 외부 호스트에서 로드된 스크립트의 무결성을 검증할 수 있어 Supply Chain 공격에 대한 방어력이 높아집니다.

```typescript
// next.config.ts
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    sri: {
      algorithm: 'sha256',
    },
  },
}

export default nextConfig
```

### Dynamic Import Tree-Shaking 강화

Dynamic Import의 Tree-Shaking이 개선됐습니다. 특정 경로에서만 조건부로 모듈을 로드하는 앱이라면, 코드 수정 없이도 Production 번들 크기가 줄어드는 효과를 기대할 수 있습니다.

```typescript
// 조건부 Dynamic Import에서 미사용 코드 경로가 더 안정적으로 제거됩니다
const AdminPanel = dynamic(
  () => import('@/components/AdminPanel'),
  { ssr: false }
);
```

## 에러 페이지 리디자인

개발용 기본 에러 페이지가 리디자인됐습니다. 스택 트레이스 가독성이 향상되고 에러 발생 위치의 코드 컨텍스트가 더 명확하게 표시됩니다. 커스텀 에러 페이지를 사용하지 않는 프로젝트에서는 업그레이드 후 자동으로 적용됩니다.

## 업그레이드 시 확인할 사항

업그레이드 후 세 가지를 확인하는 것을 권장합니다.

**Streaming Fetch 중단 문제 수정**: 특정 조건에서 스트리밍이 멈추는 버그가 16.2에서 수정됐습니다. Production에서 이 문제를 우회하고 있었다면 업그레이드로 해결될 가능성이 높습니다.

**Server Actions 변환 강화**: `node_modules` 내 Server Actions 변환이 엄격해졌습니다. Server Actions를 사용하는 서드파티 패키지와 조합할 때 예상치 못한 에러가 발생하면, 해당 패키지의 Next.js 16 지원 여부를 확인하세요.

**Turbopack 설정 이전**: `experimental.turbopack`이 Deprecated됐습니다. 설정을 최상위 `turbopack` 키로 이전하세요.

```typescript
// next.config.ts — 16.2 이상 권장 설정
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  turbopack: {
    // Turbopack 전용 설정
  },
  // experimental.turbopack 제거
}

export default nextConfig
```

## 정리

CVE-2026-29057 수정은 16.x를 Production에서 사용하고 있다면 업그레이드의 주된 이유입니다. Turbopack 개선은 일상적인 개발 속도에 직접 영향을 주고, SRI 추가는 Supply Chain 보안을 중시하는 팀에게 실질적인 강화 수단입니다. 대부분의 프로젝트에서 부담이 적은 업그레이드지만, Server Actions Route와 Turbopack 설정 이전은 반드시 확인하고 배포하세요.
